The `<SubscriptionDetailsButton />` component renders a button that opens the Subscription Details drawer when clicked. This component allows users to view and manage their subscription details, whether for their personal account or organization. The component must be rendered inside a `<SignedIn />` component to ensure the user is authenticated.

## Properties

All props are optional.

<Properties>
  - `for?`
  - `"user" | "org"`

  Determines whether to show subscription details for the current user or organization. Defaults to `"user"`.

  ---

  - `children?`
  - `React.ReactNode`

  Optional custom button element. If not provided, defaults to a button with "Subscription details" text.

  ---

  - `onSubscriptionCancel?`
  - `() => void`

  Optional callback function that is called when a subscription is cancelled.

  ---

  - `subscriptionDetailsProps?`
  - `object`

  Specify options for the underlying internal `<UserProfile />` component. For example: `<SubscriptionDetailsButton subscriptionDetailsProps={{ appearance: { ... } }} />`.
</Properties>

## Usage with frameworks

The following examples demonstrate how to use the `<SubscriptionDetailsButton />` component in different frameworks.

<Tabs items={["Next.js", "React"]}>
  <Tab>
    ```tsx {{ filename: 'app/billing/page.tsx' }}
    'use client'

    import { SignedIn } from '@clerk/nextjs'
    import { SubscriptionDetailsButton } from '@clerk/nextjs/experimental'

    export default function BillingPage() {
      return (
        <SignedIn>
          {/* Basic usage */}
          <SubscriptionDetailsButton />

          {/* Custom button for organization subscription */}
          <SubscriptionDetailsButton
            for="org"
            onSubscriptionCancel={() => console.log('Subscription canceled')}
          >
            <button className="custom-button">View Organization Subscription</button>
          </SubscriptionDetailsButton>
        </SignedIn>
      )
    }
    ```

## Requirements

The component will throw errors in the following cases:

1. When rendered outside of a `<SignedIn />` component:

```tsx
<>
  // ❌ This will throw an error
  <SubscriptionDetailsButton />
  // ✅ Correct usage
  <SignedIn>
    <SubscriptionDetailsButton />
  </SignedIn>
</>
```

2. When using `for="org"` without an active organization context:

```tsx
<>
  // ❌ This will throw an error if no organization is active
  <SubscriptionDetailsButton for="org" />
  // ✅ Correct usage
  {auth.orgId ? <SubscriptionDetailsButton for="org" /> : null}
</>
```

## Customization

The component accepts a custom button element as children, allowing you to fully customize its appearance while maintaining the subscription details functionality:

```tsx
<SubscriptionDetailsButton>
  <button className="custom-button">
    <Icon name="subscription" />
    Manage Subscription
  </button>
</SubscriptionDetailsButton>
```
