Updates the current instance.

```typescript
function update(params: UpdateParams): Promise<void>
```

## `UpdateParams`


| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="allowedorigins"></a> `allowedOrigins?` | <code>string[]</code> | For browser-like stacks such as browser extensions, Electron, or Capacitor.js, the instance allowed origins need to be updated with the request origin value. For Chrome extensions popup, background, or service worker pages the origin is `chrome-extension://extension_uiid`. For Electron apps the default origin is `http://localhost:3000`. For Capacitor.js, the origin is `capacitor://localhost`. |
| <a id="clerkjsversion"></a> `clerkJsVersion?` | <code>string \| null</code> | The npm version for `@clerk/clerk-js`. |
| <a id="developmentorigin"></a> `developmentOrigin?` | <code>string \| null</code> | The development origin for the instance. |
| <a id="enhancedemaildeliverability"></a> `enhancedEmailDeliverability?` | <code>boolean \| null</code> | Whether the instance should send emails from "verifications@clerk.dev" instead of your domain. This can be helpful if you do not have a high domain reputation. |
| <a id="hibp"></a> `hibp?` | <code>boolean \| null</code> | Whether the instance should be using the Have I Been Pwned (HIBP) service to check passwords for breaches. |
| <a id="preferredsigninstrategywhenpasswordrequired"></a> `preferredSignInStrategyWhenPasswordRequired?` | <code>"password" \| "otp" \| "" \| null</code> | Overrides the sign-in strategy that is preferred when a password is required. The value is only consulted when a password is required, and is dormant otherwise. Pass an empty string to clear the override. Passing `null` or `undefined` leaves the current value unchanged. |
| <a id="supportemail"></a> `supportEmail?` | <code>string \| null</code> | The support email for the instance. |
| <a id="testmode"></a> `testMode?` | <code>boolean \| null</code> | Toggles [test mode](/docs/guides/development/testing/test-emails-and-phones#set-up-test-mode) for this instance, allowing the use of test email addresses and phone numbers. Defaults to `true` for development instances. |
| <a id="urlbasedsessionsyncing"></a> `urlBasedSessionSyncing?` | <code>boolean \| null</code> | Whether the instance should use URL-based session syncing in development mode (i.e. without third-party cookies). |
