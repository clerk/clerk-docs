The `clerkMiddleware()` function checks the request's cookies and headers for a session JWT and makes the [`Auth`](/docs/reference/backend/types/auth-object) object available through [`getAuth()`](/docs/reference/hono/get-auth). It also attaches the Clerk Backend API client to the Hono context under the `clerk` key.

```ts
import { clerkMiddleware } from '@clerk/hono'
import { Hono } from 'hono'

const app = new Hono()

// Pass no parameters
app.use('*', clerkMiddleware())

// Pass options
app.use('*', clerkMiddleware(options))
```

## Example: Protect a route with `clerkMiddleware()` and `getAuth()`

The following example demonstrates how to use `clerkMiddleware()` and [`getAuth()`](/docs/reference/hono/get-auth) together. `clerkMiddleware()` provides authentication state to the route, and `getAuth()` is used to protect the route based on authentication status.

```ts
import { clerkMiddleware, getAuth } from '@clerk/hono'
import { Hono } from 'hono'
import { HTTPException } from 'hono/http-exception'

const app = new Hono()

app.use('*', clerkMiddleware())

app.get('/protected', (c) => {
  const auth = getAuth(c)

  if (!auth.isAuthenticated) {
    throw new HTTPException(401, { message: 'User not authenticated' })
  }

  return c.json(auth)
})

export default app
```

### Frontend API proxy

<Include src="_partials/frontend-api-proxy/explanation" />

```ts
import { clerkMiddleware } from '@clerk/hono'
import { Hono } from 'hono'

const app = new Hono()

// Enable proxy with default path ('/__clerk')
app.use(
  '*',
  clerkMiddleware({
    frontendApiProxy: {
      enabled: true,
    },
  }),
)

// Or with a custom proxy path
app.use(
  '*',
  clerkMiddleware({
    frontendApiProxy: {
      enabled: true,
      path: '/custom-clerk-proxy',
    },
  }),
)
```
