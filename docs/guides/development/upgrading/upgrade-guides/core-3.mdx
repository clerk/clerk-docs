## Overview

Core 3 is the next major version of Clerk's SDK core. This page will collect the high-level changes, release timelines, and migration expectations as they are finalized.

## What to expect (draft)

- High-impact breaking changes called out by SDK.
- Updated minimum versions for frameworks and runtimes.
- A guided migration flow with before/after examples.

## Migration flow (draft)

1. Confirm project prerequisites (framework/runtime floors).
2. Update Clerk SDK dependencies to the Core 3-compatible majors.
3. Apply breaking change fixes per SDK guide.
4. Validate UI and auth flows across protected/public routes.
