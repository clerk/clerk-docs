# Customizing your Account Portal redirects

When a user navigates to a Clerk sign up or sign in page via a link or button, Clerk will:
1. Persist the previous page's URL in a `redirect_url` query string
2. Navigate back to that page after the sign-up or sign-in is completed

This avoids interrupting the user's flow through your app. For example, a user selecting a sign in button on `example.com/foo` will navigate to `example.com/sign-in?redirect_url=example.com/foo`, then navigate back to `example.com/foo` upon completing the sign-in process.

The following sections describe the options Clerk provides for you to customize where your users are redirected to upon completing a sign-in or sign-up process.

## Environment variables

You should define the paths you want the users to be redirected to via environment variables. In this example, we redirect users to `/dashboard` after signing in, and to `/onboarding` after signing up, if the user was not already on a page that would be redirected back to.

<CodeBlockTabs type="framework" options={["Next.js", "Remix"]}>
```sh filename=".env"
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/dashboard
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/onboarding
```

```sh filename=".env"
CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/dashboard
CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/onboarding
```
</CodeBlockTabs>

If you'd like to force the redirect regardless of whether the user was on another page previously, you can use the following `FORCE` environment variable values instead:

<CodeBlockTabs type="framework" options={["Next.js", "Remix"]}>
```sh filename=".env"
NEXT_PUBLIC_CLERK_SIGN_IN_FORCE_REDIRECT_URL=/dashboard
NEXT_PUBLIC_CLERK_SIGN_UP_FORCE_REDIRECT_URL=/onboarding
```

```sh filename=".env"
CLERK_SIGN_IN_FORCE_REDIRECT_URL=/dashboard
CLERK_SIGN_UP_FORCE_REDIRECT_URL=/onboarding
```
</CodeBlockTabs>

## Middleware

If you are using Next.js and want a more programmatically generated redirect option, you can use the [`auth().protect()`](/docs/references/nextjs/auth#protect) method in your [Clerk middleware](/docs/references/nextjs/clerk-middleware).

```tsx filename="middleware.ts"
import { clerkMiddleware } from '@clerk/nextjs/server';

export default clerkMiddleware((auth, req) => {
  if (req.nextUrl.pathname.startsWith('/dashboard')) auth().protect();
});

export const config = {
  matcher: ['/((?!.*\\..*|_next).*)', '/', '/(api|trpc)(.*)'],
};
```

## Redirect URL props

You can explicitly define the redirect paths after sign-up or sign-in by passing one of the following properties to the respective components:

- `signInFallbackRedirectUrl`
- `signInForceRedirectUrl`
- `signUpFallbackRedirectUrl`
- `signUpForceRedirectUrl`

By default, when a user clicks on a link or button to navigate to a sign up or sign in page, Clerk will persist the page the user was on before into a `redirect_url` querystring, and navigate back to that page after the sign up or sign in is completed, in order to not interrupt the user's flow through your app. So, for example, if you were on `example.com/foo`, and clicked a "sign in" button on that page, it might take you to `example.com/sign-in?redirect_url=example.com/foo`. After the sign in was complete, you'd be taken back to `example.com/foo`.

This behavior can be modified using the above props, or by using [environment variables](#environment-variables). The "force redirect url" props will _always_ redirect to the provided url after sign up or sign in, regardless of what page the user was on before - it will override the `redirect_url` value if present. The "fallback redirect url" props will only redirect to the provided url if the user navigated directly to the sign up or sign in page and therefore there is no `redirect_url` value. If there is one, the fallback redirect url will not be used.

### Control components

<CodeBlockTabs options={['ClerkProvider', 'RedirectToSignIn', 'RedirectToSignUp']}>

```javascript filename="example.js"
function App() {
  return (
    <ClerkProvider
      publishableKey={clerkPubKey}
      signInFallbackRedirectUrl="/dashboard"
      signUpFallbackRedirectUrl="/onboarding"
    >
      <SignedIn>
        <Welcome />
      </SignedIn>
      <SignedOut>
        <RedirectToSignIn />
      </SignedOut>
    </ClerkProvider>
  );
}
```

```javascript filename="example.js"
function App() {
  return (
    <ClerkProvider publishableKey={clerkPubKey}>
      <SignedIn>
        <Welcome />
      </SignedIn>
      <SignedOut>
        <RedirectToSignIn
          signInFallbackRedirectUrl="/dashboard"
          signUpFallbackRedirectUrl="/onboarding"
        />
      </SignedOut>
    </ClerkProvider>
  );
}
```

```javascript filename="example.js"
function App() {
  return (
    <ClerkProvider publishableKey={clerkPubKey}>
      <SignedIn>
        <Welcome />
      </SignedIn>
      <SignedOut>
        <RedirectToSignUp
          signInFallbackRedirectUrl="/dashboard"
          signUpFallbackRedirectUrl="/onboarding"
        />
      </SignedOut>
    </ClerkProvider>
  );
}
```

</CodeBlockTabs>

<Callout type="info" emoji="ðŸ’¡">
  `<RedirectToSignIn />` or `<RedirectToSignUp />` child components will always take precedence over `<ClerkProvider>`.

### Button components

In the case that you are using a [`<SignInButton>`](/docs/components/unstyled/sign-in-button) or [`<SignUpButton>`](/docs/components/unstyled/sign-up-button), you can also pass the properties into those components. We recommend defining both `signInFallbackRedirectUrl` and `signUpFallbackRedirectUrl` redirects in each button as some users may choose to sign up instead after attempting to sign in.

<CodeBlockTabs type="framework" options={["Next.js", "React", "Remix", "Gatsby"]}>
```jsx filename="pages/index.js"
import { SignInButton, SignUpButton } from "@clerk/nextjs";

export default function Home() {
return (
  <div>
    <h1>Welcome!</h1>

    <SignInButton signInFallbackRedirectUrl="/dashboard" signUpFallbackRedirectUrl="/onboarding">
      Sign in
    </SignInButton>

    <SignUpButton signInFallbackRedirectUrl="/dashboard" signUpFallbackRedirectUrl="/onboarding">
      Sign up
    </SignUpButton>
  </div>
);
}
```

```jsx filename="example.js"
import { SignInButton, SignUpButton } from "@clerk/clerk-react";
```
