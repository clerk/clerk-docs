Clerk's components are updated regularly to improve the user experience and add new features. For most users, these updates will be rolled out automatically. This changelog highlights notable changes and new features. For a complete list of changes, see the [full changelog on GitHub](https://github.com/clerk/javascript/blob/main/packages/ui/CHANGELOG.md). If you have [specified a specific Clerk UI version](/docs/reference/components/versioning), use this changelog to understand how new releases might impact your customizations.

## 2026-05-21 - 1.13.0

### Minor Changes {{ toc: false }}

- Add an `elevation` appearance option with `raised` (default) and `flush` values. When set to `flush`, card-based components render without a border, box shadow, border radius, outer padding, or footer background, allowing them to sit flat against their container. Applies to `<SignIn />`, `<SignUp />`, `<Waitlist />`, `<CreateOrganization />`, `<OrganizationList />`, `<OAuthConsent />`, `<UserVerification />`, and session task components. Profile, popover, and modal components always render as raised, regardless of this setting.

## 2026-05-18 - 1.12.0

### Minor Changes {{ toc: false }}

- Add an `autoFocus` option to disable automatic input focusing.

### Patch Changes {{ toc: false }}

- Fix missing ARIA attributes on `MenuTrigger` and `Select` components.

## 2026-05-15 - 1.11.0

### Minor Changes {{ toc: false }}

- Add a `highlightedPlan` option to `<PricingTable />` to render a "Popular" badge on matching plans.
- Add support for inline `<bold>` markup in localization values.

## 2026-05-14 - 1.10.0

### Minor Changes {{ toc: false }}

- Add a `fontFamilyMono` option for customizing the monospace font used in Clerk components.

## 2026-05-07 - 1.9.0
