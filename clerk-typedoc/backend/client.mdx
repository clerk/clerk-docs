The Backend `Client` object is similar to the [`Client`](/docs/reference/objects/client) object as it holds information about the authenticated sessions in the current device. However, the Backend `Client` object is different from the `Client` object in that it is used in the [Backend API](/docs/reference/backend-api/model/Client){{ target: '_blank' }} and is not directly accessible from the Frontend API.

## Properties

| Property                                                             | Type                                                                                                  | Description                                                                                                  |
| -------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| <a id="createdat"></a> `createdAt`                                   | `number`                                                                                              | The Unix timestamp when the `Client` was first created.                                                      |
| <a id="id"></a> `id`                                                 | `string`                                                                                              | The unique identifier for the `Client`.                                                                      |
| <a id="lastactivesessionid"></a> `lastActiveSessionId`               | <code>null \| string</code>                                                                           | The ID of the last active [`Session`](/docs/reference/backend/types/backend-session).                        |
| <a id="lastauthenticationstrategy"></a> `lastAuthenticationStrategy` | <code>null \| [LastAuthenticationStrategy](/docs/reference/types/last-authentication-strategy)</code> | The last authentication strategy used by the `Client`.                                                       |
| <a id="sessionids"></a> `sessionIds`                                 | <code>string[]</code>                                                                                 | An array of [Session](/docs/reference/backend/types/backend-session) IDs associated with the `Client`.       |
| <a id="sessions"></a> `sessions`                                     | <code>[Session](/docs/reference/backend/types/backend-session)[]</code>                               | An array of [`Session`](/docs/reference/backend/types/backend-session) objects associated with the `Client`. |
| <a id="signinid"></a> `signInId`                                     | <code>null \| string</code>                                                                           | The ID of the associated [`SignIn`](/docs/reference/objects/sign-in).                                        |
| <a id="signupid"></a> `signUpId`                                     | <code>null \| string</code>                                                                           | The ID of the associated [`SignUp`](/docs/reference/objects/sign-up).                                        |
| <a id="updatedat"></a> `updatedAt`                                   | `number`                                                                                              | The Unix timestamp when the `Client` was last updated.                                                       |
