The fastest way to add Clerk to Next.js is the [Clerk CLI](/docs/cli). A single `clerk init` command installs the [Clerk Next.js SDK](/docs/reference/nextjs/overview), connects your project to a Clerk application, and scaffolds the middleware, provider, and sign-in and sign-up pages you need to start authenticating users. If you don't have a Next.js app yet, `clerk init` can create one for you.

> [!TIP]
> Prefer to wire things up yourself? Follow the [manual setup guide](/docs/getting-started/quickstart/manual) instead.

<Steps>
  ## Run `clerk init`

  Run the following command in your terminal:

  ```npm
  npx clerk@latest init
  ```

  The CLI walks you through setup interactively:

  - If you're not already in a Next.js project, it offers to create one for you.
  - It signs you in to Clerk in your browser and connects your project to a Clerk application.
  - It installs the `@clerk/nextjs` SDK and scaffolds Clerk into your app.

  > [!TIP]
  > Want to start without signing in? Run `clerk init --keyless` to scaffold your app with temporary development keys, then run `clerk auth login` when you're ready to connect a Clerk account.

  ## Review what `clerk init` set up

  When `clerk init` finishes, your app is ready to authenticate users. Here's what it added:

  - **`.env.local`** — your Clerk [publishable and secret keys](/docs/guides/development/clerk-environment-variables), pulled from the application you connected to.
  - **`proxy.ts`** (or `middleware.ts` on Next.js 15 and earlier) — adds [`clerkMiddleware()`](/docs/reference/nextjs/clerk-middleware) with route protection. By default, every route except `/sign-in` and `/sign-up` requires the user to be signed in. Edit the `isPublicRoute` matcher to change which routes are public.
  - **`app/layout.tsx`** — wraps your app's root layout in [`<ClerkProvider>`](/docs/reference/components/clerk-provider), which makes Clerk's [components](/docs/reference/components/overview) and hooks available throughout your app. When `clerk init` creates a new app, it also adds a header with Clerk's [`<SignInButton>`](/docs/reference/components/unstyled/sign-in-button), [`<SignUpButton>`](/docs/reference/components/unstyled/sign-up-button), and [`<UserButton>`](/docs/reference/components/user/user-button).
  - **`app/sign-in` and `app/sign-up`** — dedicated pages rendering Clerk's [`<SignIn />`](/docs/reference/components/authentication/sign-in) and [`<SignUp />`](/docs/reference/components/authentication/sign-up) components.
