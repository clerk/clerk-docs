The Backend `APIKey` object holds information about an API key.

## Properties

| Property                                         | Type                                       | Description                                                                       |
| ------------------------------------------------ | ------------------------------------------ | --------------------------------------------------------------------------------- |
| <a id="claims"></a> `claims`                     | <code>null \| Record\<string, any\></code> | Custom claims associated with the API key.                                        |
| <a id="createdat"></a> `createdAt`               | `number`                                   | The Unix timestamp when the API key was created.                                  |
| <a id="createdby"></a> `createdBy`               | <code>null \| string</code>                | The user ID for the user creating the API key.                                    |
| <a id="description"></a> `description`           | <code>null \| string</code>                | A description for the API key.                                                    |
| <a id="expiration"></a> `expiration`             | <code>null \| number</code>                | The Unix timestamp when the API key expires. `null` if the API key never expires. |
| <a id="expired"></a> `expired`                   | `boolean`                                  | Whether the API key has expired.                                                  |
| <a id="id"></a> `id`                             | `string`                                   | The unique identifier for the API key.                                            |
| <a id="lastusedat"></a> `lastUsedAt`             | <code>null \| number</code>                | The Unix timestamp when the API key was last used to authenticate a request.      |
| <a id="name"></a> `name`                         | `string`                                   | The API key's name.                                                               |
| <a id="revocationreason"></a> `revocationReason` | <code>null \| string</code>                | The reason for revoking the API key, if it has been revoked.                      |
| <a id="revoked"></a> `revoked`                   | `boolean`                                  | Whether the API key has been revoked.                                             |
| <a id="scopes"></a> `scopes`                     | <code>string[]</code>                      | An array of scopes that define what the API key can access.                       |
| <a id="secret"></a> `secret?`                    | `string`                                   | The API key secret.                                                               |
| <a id="subject"></a> `subject`                   | `string`                                   | The user or organization ID that the API key is associated with.                  |
| <a id="type"></a> `type`                         | `string`                                   | The type of the API key. Currently always `'api_key'`.                            |
| <a id="updatedat"></a> `updatedAt`               | `number`                                   | The Unix timestamp when the API key was last updated.                             |
