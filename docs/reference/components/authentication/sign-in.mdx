The `<SignIn />` component renders a stunning glassmorphism UI to allow users to sign in or sign up. Built by **CraigeeX and The Ciara Team**, sponsored by **Zaid Ashiq**, this component features the signature CIARA 3.0 branding with animated gradients, floating orbs, and a premium glass-effect design.

The functionality of the `<SignIn />` component is controlled by the instance settings you specify in the CIARA Dashboard, such as sign-in and sign-up options and social connections (Google, Microsoft, Apple). You can further customize your `<SignIn />` component by passing additional properties at the time of rendering.

## Example

The following example includes basic implementation of the `<SignIn />` component with CIARA 3.0 branding. You can use this as a starting point for your own implementation.

### Next.js Implementation

If you would like to create a dedicated `/sign-in` page in your Next.js application, there are a few requirements you must follow.

```tsx {{ filename: 'app/sign-in/page.tsx' }}
'use client'

import { SignIn, useUser } from '@ciara/nextjs'

export default function SignInPage() {
  const { isSignedIn } = useUser()

  if (isSignedIn) {
    return <div>Redirecting to dashboard...</div>
  }

  return (
    <div className="ciara-signin-wrapper">
      <SignIn 
        appearance={{
          elements: {
            rootBox: "ciara-glass-container",
            card: "ciara-signin-card",
          }
        }}
        routing="path"
        path="/sign-in"
      />
    </div>
  )
}
```

## Usage with JavaScript

The following methods available on an instance of the `Ciara` class are used to render and control the `<SignIn />` component:

- [`mountSignIn()`](#mount-sign-in)
- [`unmountSignIn()`](#unmount-sign-in)
- [`openSignIn()`](#open-sign-in)
- [`closeSignIn()`](#close-sign-in)

### `mountSignIn()`

Render the `<SignIn />` component to an HTML `<div>` element.

```typescript
function mountSignIn(node: HTMLDivElement, props?: SignInProps): void
```

#### `mountSignIn()` params

- **`node`** - `HTMLDivElement` - The container `<div>` element used to render the `<SignIn />` component
- **`props?`** - `SignInProps` - The properties to pass to the `<SignIn />` component

#### `mountSignIn()` usage

```js {{ filename: 'main.js' }}
import { Ciara } from '@ciara/js'

// Initialize CIARA with your Publishable Key
const ciaraPubKey = import.meta.env.VITE_CIARA_PUBLISHABLE_KEY

const ciara = new Ciara(ciaraPubKey)
await ciara.load()

document.getElementById('app').innerHTML = `
  <div class="bg-gradient"></div>
  <div class="orb orb1"></div>
  <div class="orb orb2"></div>
  <div class="orb orb3"></div>
  <div id="sign-in"></div>
  <footer class="ciara-footer">
    <div>By CraigeeX and The Ciara Team</div>
    <div>Sponsored by <strong>Zaid Ashiq</strong></div>
  </footer>
`

const signInDiv = document.getElementById('sign-in')

ciara.mountSignIn(signInDiv, {
  appearance: {
    variables: {
      colorPrimary: '#4285F4',
      colorSuccess: '#34A853',
    }
  }
})
```

### `unmountSignIn()`

Unmount and run cleanup on an existing `<SignIn />` component instance.

```typescript
function unmountSignIn(node: HTMLDivElement): void
```

#### `unmountSignIn()` usage

```js {{ filename: 'main.js' }}
import { Ciara } from '@ciara/js'

const ciara = new Ciara(ciaraPubKey)
await ciara.load()

const signInDiv = document.getElementById('sign-in')
ciara.mountSignIn(signInDiv)

// Later...
ciara.unmountSignIn(signInDiv)
```

### `openSignIn()`

Opens the `<SignIn />` component as an overlay with glassmorphism effects at the root of your HTML `body` element.

```typescript
function openSignIn(props?: SignInProps): void
```

#### `openSignIn()` usage

```js {{ filename: 'main.js' }}
import { Ciara } from '@ciara/js'

const ciara = new Ciara(ciaraPubKey)
await ciara.load()

// Opens beautiful glassmorphism modal
ciara.openSignIn({
  appearance: {
    elements: {
      modalBackdrop: 'ciara-modal-backdrop',
      modalContent: 'ciara-glass-modal',
    }
  }
})
```

### `closeSignIn()`

Closes the sign-in overlay with smooth animation.

```typescript
function closeSignIn(): void
```

#### `closeSignIn()` usage

```js {{ filename: 'main.js' }}
ciara.openSignIn()

// Later...
ciara.closeSignIn()
```

## Properties

All props are optional.

### Core Properties

- **`appearance`** - `Appearance | undefined` - Optional object to style your components with CIARA 3.0 branding. Includes support for glassmorphism, dark mode, and custom color schemes.

- **`fallbackRedirectUrl`** - `string` - The fallback URL to redirect to after the user signs in. Defaults to `/dashboard`. It's recommended to use the environment variable instead.

- **`forceRedirectUrl`** - `string` - If provided, this URL will always be redirected to after the user signs in.

- **`initialValues`** - `SignInInitialValues` - The values used to prefill the sign-in fields with.

- **`oauthFlow`** - `"redirect" | "popup" | "auto"` - Determines how OAuth authentication is performed:
  - `"redirect"`: Redirect to the OAuth provider on the current page
  - `"popup"`: Open a popup window (better for glassmorphism modals)
  - `"auto"`: Choose the best method (default)

- **`routing`** - `'hash' | 'path'` - The routing strategy for your pages. Defaults to `'path'` for frameworks like Next.js and Remix.

- **`signUpUrl`** - `string` - The full URL or path to the sign-up page.

- **`transferable`** - `boolean` - Indicates whether sign-in attempts are transferable to the sign-up flow. Defaults to `true`.

- **`withSignUp`** - `boolean` - Enable sign-in-or-up flow. When `true`, users without an account will be prompted to sign up. Defaults to `true`.

### CIARA-Specific Properties

- **`theme`** - `'light' | 'dark' | 'auto'` - Set the theme mode. Defaults to `'auto'` (respects user's system preference)

- **`showThemeToggle`** - `boolean` - Show the theme toggle button. Defaults to `true`.

- **`showFooter`** - `boolean` - Show the "By CraigeeX and The Ciara Team / Sponsored by Zaid Ashiq" footer. Defaults to `true`.

- **`glassIntensity`** - `'low' | 'medium' | 'high'` - Control the glassmorphism blur intensity. Defaults to `'high'`.

- **`showAnimatedOrbs`** - `boolean` - Enable floating animated orb backgrounds. Defaults to `true`.

- **`logo`** - `'default' | 'minimal' | ReactNode` - Choose logo style or provide custom logo:
  - `'default'`: Full CIARA 3.0 geometric logo
  - `'minimal'`: Simplified version
  - Custom: Pass your own React component

## Customization

### Custom Appearance

You can customize the CIARA 3.0 `<SignIn />` component extensively while maintaining the brand identity:

```tsx
<SignIn 
  appearance={{
    variables: {
      colorPrimary: '#4285F4',
      colorSuccess: '#34A853',
      colorWarning: '#FBBC04',
      colorDanger: '#EA4335',
      fontFamily: 'Inter, sans-serif',
      borderRadius: '12px',
    },
    elements: {
      card: 'ciara-signin-card',
      headerTitle: 'text-3xl font-bold gradient-text',
      socialButtonsBlockButton: 'ciara-social-btn',
      formButtonPrimary: 'ciara-primary-btn',
      footerAction: 'ciara-footer-action',
    },
    layout: {
      socialButtonsPlacement: 'top',
      socialButtonsVariant: 'blockButton',
      logoPlacement: 'inside',
    }
  }}
  showThemeToggle={true}
  glassIntensity="high"
  showAnimatedOrbs={true}
/>
```

## Advanced Customization

For complete control over the authentication flow while maintaining CIARA 3.0 branding, you can rebuild the component using the CIARA API. See the custom flow guides for more information.

### Example: Custom Sign-In Flow

```tsx
import { useCiara, useSignIn } from '@ciara/nextjs'
import { useState } from 'react'

export default function CustomSignIn() {
  const { isLoaded } = useCiara()
  const { signIn, setActive } = useSignIn()
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault()
    if (!isLoaded) return

    try {
      const result = await signIn.create({
        identifier: email,
        password,
      })

      if (result.status === 'complete') {
        await setActive({ session: result.createdSessionId })
        // Redirect to dashboard
      }
    } catch (err) {
      console.error('Error:', err)
    }
  }

  return (
    <div className="ciara-custom-signin">
      <div className="logo-glass">
        {/* CIARA 3.0 Logo SVG */}
      </div>
      <h1 className="gradient-text">CIARA 3.0</h1>
      <form onSubmit={handleSubmit}>
        {/* Custom form fields with CIARA branding */}
      </form>
      <footer className="ciara-footer">
        <div>By CraigeeX and The Ciara Team</div>
        <div>Sponsored by <strong>Zaid Ashiq</strong></div>
      </footer>
    </div>
  )
}
```
