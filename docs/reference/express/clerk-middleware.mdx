The `clerkMiddleware()` function checks the request's cookies and headers for a session JWT and if found, attaches the [`Auth`](/docs/reference/backend/types/auth-object) object to the `request` object under the `auth` key. **It must be set before any other middleware.**

> [!TIP]
> Use `clerkMiddleware()` with [`getAuth()`](/docs/reference/express/get-auth) to protect routes while keeping control over unauthenticated responses.

```js
import { clerkMiddleware } from '@clerk/express'

const app = express()

// Pass no parameters
app.use(clerkMiddleware())

// Pass options
app.use(clerkMiddleware(options))
```

## Example: Use `clerkMiddleware()` and `getAuth()` together

The following example demonstrates how to use `clerkMiddleware()` and [`getAuth()`](/docs/reference/express/get-auth) together. `clerkMiddleware()` provides authentication state to your routes, and `getAuth()` protects a route based on authentication and authorization status.

```js
import { clerkMiddleware, getAuth } from '@clerk/express'
import express from 'express'

const app = express()
const PORT = 3000

// Apply `clerkMiddleware()` to all routes
app.use(clerkMiddleware())

app.get('/path', (req, res) => {
  const auth = getAuth(req)

  // Handle if the user is not authenticated
  if (!auth.userId) {
    return res.status(401).send('Unauthorized')
  }

  // Handle if the user is authenticated but not authorized
  if (!auth.has({ permission: 'org:admin:example' })) {
    return res.status(403).send('Forbidden')
  }

  return res.json(auth)
})

// This route is not protected but it will have authentication state
// attached to the request object because `clerkMiddleware()` was applied to all routes
app.get('/path2', (req, res) => res.json(getAuth(req)))

// Start the server and listen on the specified port
app.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`)
})
```

### `clerkMiddleware()` options

<Include src="_partials/clerk-middleware-options" />

<Properties>
  - `clerkClient`
  - [`ClerkClient`](/docs/reference/backend/overview#create-clerk-client-options)

  An instance of the `ClerkClient` class. This is used to interact with the Clerk API.

  ---

  - `debug`
  - `boolean`

  A flag to enable debug mode. When set to `true`, the middleware will log debug information to the console. Defaults to `false`.

### Frontend API proxy

<Include src="_partials/frontend-api-proxy/explanation" />

```js
import { clerkMiddleware } from '@clerk/express'
import express from 'express'

const app = express()

// Enable proxy with default path ('/__clerk')
app.use(
  clerkMiddleware({
    frontendApiProxy: {
      enabled: true,
    },
  }),
)

// Or with a custom proxy path
app.use(
  clerkMiddleware({
    frontendApiProxy: {
      enabled: true,
      path: '/custom-clerk-proxy',
    },
  }),
)
```
