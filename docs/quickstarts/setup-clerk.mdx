## You will learn how to

* Create a Clerk account
* Create a new Clerk project

**Estimated time** 1 cup of coffee

<Callout type="info">
  Migrating from another platform?

  You might want to [check out one of our migration guides](/docs/deployments/migrate-overview) to help
  you move your data to Clerk.
</Callout>

<Steps>

### Sign into Clerk

[Create a Clerk account](https://dashboard.clerk.com/sign-up) or [sign into your Clerk dasbhoard](https://dashboard.clerk.com/).

### Create a Clerk application

If you've just created an account for the first time, you'll be taken directly to our interactive authentication setup form.

{/* Don't move ![] to a new line, otherwise MDX adds a <p> tag and it breaks A11Y */}

<div style={{filter: "drop-shadow(rgba(0, 0, 0, 0.16) 0px 12px 24px)"}}>![An interactive builder to make your "<SignIn />" component truly yours, which lets you select your auth providers.](/docs/images/setup-clerk/create-sign-in-comp.png)</div>

Otherwise, you'll be redirected to [your Clerk dashboard](https://dashboard.clerk.com/).

To create a new app, press the **Add application** card to be taken to the interactive authentication setup form.

{/* Don't move ![] to a new line, otherwise MDX adds a <p> tag and it breaks A11Y */}

<div style={{filter: "drop-shadow(rgba(0, 0, 0, 0.16) 0px 12px 24px)", maxWidth: "400px"}}>![An "Add application" card on your dashboard](/docs/images/setup-clerk/add-application.png)</div>

### Select auth providers

Clerk provides [over 20 authentication providers](/docs/authentication/social-connections/overview) that you can use in your Clerk apps. Select the ones that matter to your users and press **Create application.**

Once the application is created in the Clerk dashboard, you will be redirected to the **[API Keys](https://dashboard.clerk.com/last-active?path=api-keys)** page, which will show you your application's API keys.
