> [!TIP]
> **Protect access as close to the resource as possible**, in the page, route handler, Server Action, or other server-side code that reads or mutates the data. The same code that touches the data also enforces who can reach it, so authorization stays next to the thing it protects. See [Protect pages and routes](/docs/guides/secure/protect-pages).
>
> `clerkMiddleware()` is required, and it's the right place for redirects, headers, and other request handling.
