This flow handles the entire OAuth sign-in process for you, from provider selection to user authentication, with a production-ready UI that’s already wired to Clerk’s backend.

<InteractiveOAuthSignIn title="Try OAuth Sign-in" />

## Quick install

Creates a new Next.js project with Clerk preconfigured for OAuth sign-in, your selected styling, and the correct routes.

```bash {{ prettier: false }}
npx clerk-cli create-flow sign-in-oauth \
  --framework=nextjs \
  --integration=components \
  --styling=default \
```

## Project

View the full project code for this example flow, using Clerk’s Prebuilt UI components.

```tsx {{ prettier: false }}
import { SignIn } from '@clerk/nextjs'

export default function SignInPage() {
  return (
    <div className="flex justify-center items-center min-h-screen">
      <SignIn 
        appearance={{
          elements: {
            formButtonPrimary: 'bg-blue-600 hover:bg-blue-700 text-sm normal-case'
          }
        }}
        fallbackRedirectUrl="/"
        forceRedirectUrl="/"
      />
    </div>
  )
}
```

## Before you start

Before you install this flow, make sure you have:

- A Clerk application in the Dashboard
- At least one OAuth provider enabled (e.g., Google, GitHub, Microsoft) - Set up under the Social Connections section
- Frontend API key and Secret Key available (You'll find these in the API Keys section of the Clerk Dashboard)
- Node.js 16+ and your preferred framework configured
- Make sure you've followed your framework's setup guide

## Usage

Insert the sign-in flow where it belongs in your app and adjust redirects and styling as needed.

```tsx {{ prettier: false }}
import { SignIn } from '@clerk/nextjs'
import { dark } from '@clerk/themes'

export default function Page() {
  return (
    <SignIn
      appearance={{
        baseTheme: dark,
        elements: {
          formButtonPrimary: 'bg-slate-500 hover:bg-slate-600 text-sm',
          card: 'bg-slate-800'
        }
      }}
      fallbackRedirectUrl="/dashboard"
      forceRedirectUrl="/onboarding"
    />
  )
}
```
