# Sign-up

## Email/Password + Phone verification

In your [Email, Phone, Username section](https://dashboard.clerk.com/last-active?path=user-authentication/email-phone-username) of the dashboard only the Email address and Password would be activated for this example. In the multi-factor section the SMS verification code would be activated.

```tsx {{ filename: '/app/sign-up/[[...sign-up]]/page.tsx', collapsible: true }}
"use client";

import * as Clerk from "@clerk/elements/common";
import * as SignUp from "@clerk/elements/sign-up";

export default function SignUpPage() {
  return (
    <SignUp.Root>
      <SignUp.Step name="start">
        <Clerk.GlobalError />
        <Clerk.Field name="identifier">
          <Clerk.Label>Email</Clerk.Label>
          <Clerk.Input type="email" required />
          <Clerk.FieldError />
        </Clerk.Field>
        <Clerk.Field name="password">
          <Clerk.Label>Password</Clerk.Label>
          <Clerk.Input type="password" required />
          <Clerk.FieldError />
        </Clerk.Field>
        <SignUp.Action submit>Sign Up</SignUp.Action>
      </SignUp.Step>
      <SignUp.Step name="verifications">
        <Clerk.GlobalError />
        <SignUp.Strategy name="phone_code">
          <Clerk.Field name="code">
            <Clerk.Label>Phone code</Clerk.Label>
            <Clerk.Input type="otp" required />
            <Clerk.FieldError />
          </Clerk.Field>
          <SignUp.Action submit>Finish registration</SignUp.Action>
        </SignUp.Strategy>
      </SignUp.Step>
    </SignUp.Root>
  );
}
```

## Email/Password + Username + Email verification

On the first screen (with `<Step name="start">`) a user inputs the email and password. On the second screen a username is asked for. Once that information is provided the sign-up process is ended with an email verification to the user's email address.

In your [Email, Phone, Username section](https://dashboard.clerk.com/last-active?path=user-authentication/email-phone-username) of the dashboard you can decide which identifiers are required for sign-up. In the below example youâ€™d need to make username required so that the `<Step name="continue">` screen is rendered. Otherwise it would be optional and skipped.


```tsx {{ filename: '/app/sign-up/[[...sign-up]]/page.tsx', collapsible: true }}
"use client";

import * as Clerk from "@clerk/elements/common";
import * as SignUp from "@clerk/elements/sign-up";

export default function SignUpPage() {
  return (
    <SignUp.Root>
      <SignUp.Step name="start">
        <Clerk.GlobalError />
        <Clerk.Field name="emailAddress">
          <Clerk.Label>Email</Clerk.Label>
          <Clerk.Input type="email" required />
          <Clerk.FieldError />
        </Clerk.Field>
        <Clerk.Field name="password">
          <Clerk.Label>Password</Clerk.Label>
          <Clerk.Input type="password" required />
          <Clerk.FieldError />
        </Clerk.Field>
        <SignUp.Action submit>Sign Up</SignUp.Action>
      </SignUp.Step>
      <SignUp.Step name="continue">
        <Clerk.GlobalError />
        <Clerk.Field name="username">
          <Clerk.Label>Username</Clerk.Label>
          <Clerk.Input type="text" required />
          <Clerk.FieldError />
        </Clerk.Field>
        <SignUp.Action submit>Continue registration</SignUp.Action>
      </SignUp.Step>
      <SignUp.Step name="verifications">
        <Clerk.GlobalError />
        <SignUp.Strategy name="email_code">
          <Clerk.Field name="code">
            <Clerk.Label>Email code</Clerk.Label>
            <Clerk.Input type="otp" required />
            <Clerk.FieldError />
          </Clerk.Field>
          <SignUp.Action submit>Finish registration</SignUp.Action>
        </SignUp.Strategy>
      </SignUp.Step>
    </SignUp.Root>
  );
}
```
