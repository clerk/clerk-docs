description: Clerk's OrganizationListView renders a standalone account picker for choosing a personal or organization account.
sdk: ios, android
---

<If sdk="android">
  ![The OrganizationListView displays a switch account sheet with the active organization selected, other organizations, and a create organization action.](/docs/images/ui-components/android-organization-list-view.png){{ style: { maxWidth: '427px' } }}
</If>

The `OrganizationListView` renders a standalone account picker for choosing a personal or organization account. It lists the personal account when allowed, organization memberships, pending organization invitations, suggested organizations, and a create-organization action when organization creation is enabled for the current user.

<If sdk="ios">
  When the user selects a personal account or organization, `OrganizationListView` calls `clerk.auth.setActive(sessionId:organizationId:)` internally. Personal account selection clears the [Active Organization](!active-organization). Organization selection sets the selected organization as active.
</If>

<If sdk="android">
  When the user selects a personal account or organization, `OrganizationListView` calls `Clerk.auth.setActive(...)` internally. Personal account selection passes `null` to `onAccountSelected`; organization selection passes the selected organization ID.
</If>

## Parameters

<If sdk="ios">
  <Properties>
    - `hidePersonal`
    - `Bool`

    Whether the personal account row should be hidden even when personal account selection is allowed. Defaults to `false`.

    ---

    - `isDismissable`
    - `Bool`

    Whether the view can be dismissed by the user. When `true`, a dismiss button appears and the view closes automatically after account selection. When `false`, no dismiss button is shown. Defaults to `true`.

    ---

    - `navigationPath`
    - `Binding<NavigationPath>?`

    An optional parent navigation path for embedded usage. Pass a parent path when the view is hosted inside your own `NavigationStack`.

    ---

    - `skipInvitationScreen`
    - `Bool`

    Whether creating an organization should skip the post-create invite step. Defaults to `false`.
  </Properties>
</If>

<If sdk="android">
  <Properties>
    - `modifier`
    - `Modifier`

    A modifier that gets applied to the `OrganizationListView` composable.

    ---

    - `clerkTheme`
    - `ClerkTheme?`

    The theme to apply to the `OrganizationListView`.

    ---

    - `hidePersonalAccount`
    - `Boolean`

    Whether to hide the personal account row even when personal account selection is allowed. Defaults to `false`.

    ---

    - `isDismissable`
    - `Boolean`

    Whether to show a top dismiss affordance and call `onDismissRequest` after a successful selection when possible. Defaults to `true`.

    ---

    - `skipPostCreateInviteFlow`
    - `Boolean`

    Reserved for create-organization flows that include a post-create invitation step. Android's current create flow already stops after creation. Defaults to `false`.

    ---

    - `onDismissRequest`
    - `(() -> Unit)?`

    Called when the dismiss affordance is pressed or when a dismissable selection completes.

    ---

    - `onCreateOrganization`
    - `(OrganizationCreationDefaults?) -> Unit`

    Called when the create-organization row is selected. The latest creation defaults are provided when available.

    ---

    - `onAccountSelected`
    - `((String?) -> Unit)?`

    Called after selecting the personal account or an organization. Receives `null` for personal account selection and the organization ID for organization selection.
  </Properties>
</If>

## Usage

### Fullscreen account picker

<If sdk="ios">
  The following examples show how to use the `OrganizationListView` in your iOS app.

  ```swift
  import ClerkKitUI
  import SwiftUI

  struct AccountPickerView: View {
    var body: some View {
      OrganizationListView(isDismissable: false)
    }
  }
  ```

  ### Dismissible sheet

  ```swift
  import ClerkKitUI
  import SwiftUI

  struct AccountPickerButton: View {
    @State private var accountPickerIsPresented = false

    var body: some View {
      Button("Switch account") {
        accountPickerIsPresented = true
      }
      .sheet(isPresented: $accountPickerIsPresented) {
        OrganizationListView()
      }
    }
  }
  ```

  ### Embedded navigation

  ```swift
  import ClerkKitUI
  import SwiftUI

  struct AccountPickerScreen: View {
    @State private var path = NavigationPath()

    var body: some View {
      NavigationStack(path: $path) {
        OrganizationListView(
          isDismissable: false,
          navigationPath: $path
        )
      }
    }
  }
  ```
</If>

<If sdk="android">
  The following examples show how to use the `OrganizationListView` in your Android app.

  ```kotlin
  import androidx.compose.foundation.layout.fillMaxSize
  import androidx.compose.runtime.Composable
  import androidx.compose.ui.Modifier
  import com.clerk.ui.organizationlist.OrganizationListView

  @Composable
  fun AccountPickerScreen(onDone: () -> Unit) {
      OrganizationListView(
          modifier = Modifier.fillMaxSize(),
          onDismissRequest = onDone,
          onAccountSelected = { selectedOrganizationId ->
              // selectedOrganizationId is null when the user selects their personal account.
              onDone()
          },
      )
  }
  ```

  ### Custom create organization flow

  ```kotlin
  import com.clerk.api.organizations.OrganizationCreationDefaults
  import com.clerk.ui.organizationlist.OrganizationListView

  OrganizationListView(
      onCreateOrganization = { creationDefaults: OrganizationCreationDefaults? ->
          // Navigate to your own create organization flow.
      },
  )
  ```
</If>

### Hide personal account selection

<If sdk="ios">
  ```swift
  OrganizationListView(hidePersonal: true)
  ```
</If>

<If sdk="android">
  ```kotlin
  OrganizationListView(
      hidePersonalAccount = true,
      onAccountSelected = { organizationId ->
          // organizationId is non-null when personal account selection is hidden.
      },
  )
  ```
</If>
