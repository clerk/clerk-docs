The `getAuth()` helper retrieves authentication state from the Hono context. It must be used after [`clerkMiddleware()`](/docs/reference/hono/clerk-middleware) has been applied.

## Usage

The `getAuth()` helper can be used to protect routes based on authentication status, authorization status, and token type. It also offers granular control over how to handle unauthenticated users - you can return a `401` status code, redirect them, or perform whatever action you need.

### Example: Protect a route based on authentication status

The following example uses `getAuth()` to protect the route based on _authentication_ status.

```ts
import { clerkMiddleware, getAuth } from '@clerk/hono'
import { Hono } from 'hono'
import { HTTPException } from 'hono/http-exception'

const app = new Hono()

app.use('*', clerkMiddleware())

app.get('/protected', (c) => {
  const auth = getAuth(c)

  if (!auth.isAuthenticated) {
    throw new HTTPException(401, { message: 'User not authenticated' })
  }

  return c.json(auth)
})

export default app
```

### Example: Protect a route based on authorization status

The following example uses `getAuth()` to protect the route based on _authorization_ status.

```ts
import { clerkMiddleware, getAuth } from '@clerk/hono'
import { Hono } from 'hono'

const app = new Hono()

app.use('*', clerkMiddleware())

app.get('/admin', (c) => {
  const auth = getAuth(c)

  if (!auth.has({ permission: 'org:admin:example' })) {
    return c.text('Forbidden', 403)
  }

  return c.json(auth)
})

export default app
```

For more examples on how to use `getAuth()` to perform [authorization checks](!authorization-check), see the [dedicated guide](/docs/guides/secure/authorization-checks).

### Example: Protect a route based on token type

The following example uses `getAuth()` to protect the route based on token type:

- It accepts any token type `(acceptsToken: 'any')` from the request.
- If the token is a `session_token`, it logs that the request is from a user session.
- Otherwise, it logs that the request uses a machine token and specifies its type.

```ts
import { clerkMiddleware, getAuth } from '@clerk/hono'
import { Hono } from 'hono'

const app = new Hono()

app.use('*', clerkMiddleware())

app.get('/path', (c) => {
  const authObject = getAuth(c, { acceptsToken: 'any' })

  if (authObject.tokenType === 'session_token') {
    console.log('This is a session token from a user')
  } else {
    console.log(`This is a ${authObject.tokenType} token`)
  }

  return c.json(authObject)
})

export default app
```

## `getAuth()` options

<Properties>
  - `context`

  The Hono context object.

  ---

  - `opts?`
  - `{acceptsToken: TokenType, treatPendingAsSignedOut: boolean }`

  An object that can be used to configure the behavior of the `getAuth()` function. It accepts the following properties:

  <Include src="_partials/machine-auth/accepts-token-explanation" />

  - `treatPendingAsSignedOut?`: A boolean that indicates whether to treat [`pending` session status](/docs/reference/types/session-status#properties) as signed out. Defaults to `true`.
</Properties>
