Updates the metadata for the given user, by merging existing values with the provided parameters.

A "deep" merge will be performed - "deep" means that any nested JSON objects will be merged as well. You can remove metadata keys at any level by setting their value to `null`.

> [!TIP]
> If you want to fully replace the existing metadata instead of merging, use [`replaceUserMetadata()`](/docs/reference/backend/user/replace-user-metadata).

Returns the updated [`User`](/docs/reference/backend/types/backend-user).

```typescript
function updateUserMetadata(userId: string, params: { privateMetadata?: UserPrivateMetadata; publicMetadata?: UserPublicMetadata; unsafeMetadata?: UserUnsafeMetadata }): Promise<User>
```

## Parameters


| Parameter | Type | Description |
| ------ | ------ | ------ |
| `userId` | `string` | The ID of the user to update. |
| `params` | <code>\{ privateMetadata?: [UserPrivateMetadata](/docs/reference/types/metadata#user-private-metadata); publicMetadata?: [UserPublicMetadata](/docs/reference/types/metadata#user-public-metadata); unsafeMetadata?: [UserUnsafeMetadata](/docs/reference/types/metadata#user-unsafe-metadata); \}</code> | The metadata to update. |
| `params.privateMetadata?` | [UserPrivateMetadata](/docs/reference/types/metadata#user-private-metadata) | Metadata that can be read and set only from the [Backend API](/docs/reference/backend-api){{ target: '_blank' }}. |
| `params.publicMetadata?` | [UserPublicMetadata](/docs/reference/types/metadata#user-public-metadata) | Metadata that can be read from the Frontend API and [Backend API](/docs/reference/backend-api){{ target: '_blank' }} and can be set only from the Backend API. |
| `params.unsafeMetadata?` | [UserUnsafeMetadata](/docs/reference/types/metadata#user-unsafe-metadata) | Metadata that can be read and set from the Frontend API. It's considered unsafe because it can be modified from the Frontend API. |
