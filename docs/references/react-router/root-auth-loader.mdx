---
title: '`rootAuthLoader()`'
description: The rootAuthLoader() is used to make Clerk's authentication state accessible.
---

The `rootAuthLoader()` function configures Clerk to handle authentication state for React Router routes, allowing you to easily access user session information within your app.

## Usage

Inside `root.tsx` add `rootAuthLoader()` to the `loader` function. If it doesn't exist yet you'll have to add it manually.

```tsx {{ filename: 'app/root.tsx' }}
import { rootAuthLoader } from '@clerk/react-router/ssr.server'
import type { Route } from './+types/root'

export async function loader(args: Route.LoaderArgs) {
  return rootAuthLoader(args)
}
```

### Pass options

Optionally, you can pass options to the `rootAuthLoader()` as the second argument.

```tsx {{ filename: 'app/root.tsx', mark: [5] }}
import { rootAuthLoader } from '@clerk/react-router/ssr.server'
import type { Route } from './+types/root'

export async function loader(args: Route.LoaderArgs) {
  return rootAuthLoader(args, { signInUrl: '/sign-in' })
}
```

### Callback

If you need to load additional data, you can pass your loader directly to `rootAuthLoader()` as the second argument. The options will be the third argument then.

```tsx {{ filename: 'app/root.tsx', mark: [7, 12] }}
import { rootAuthLoader } from '@clerk/react-router/ssr.server'
import type { Route } from './+types/root'

export async function loader(args: Route.LoaderArgs) {
  return rootAuthLoader(
    args,
    ({ request, context, params }) => {
      const { userId } = request.auth

      return { userId }
    },
    { signInUrl: '/sign-in' },
  )
}
```
