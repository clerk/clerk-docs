The fastest way to add Clerk to a Next.js app is with the [Clerk CLI](/docs/cli). Running `clerk init` detects your framework, installs the [Clerk Next.js SDK](/docs/reference/nextjs/overview), connects your project to a Clerk application, and scaffolds everything you need to start authenticating users.

> [!TIP]
> Prefer to wire things up yourself? Follow the [manual setup guide](/docs/getting-started/quickstart/manual) instead.

<Steps>
  ## Create a new Next.js app

  If you don't already have a Next.js app, run the following commands to [create a new one](https://nextjs.org/docs/getting-started/installation).

  ```npm
  npm create next-app@latest clerk-nextjs -- --yes
  cd clerk-nextjs
  npm install
  ```

  ## Run `clerk init`

  In your project directory, run `clerk init`. The CLI prompts you to sign in to (or create) a Clerk account in your browser, then connects your project to a Clerk application.

  ```npm
  npx clerk init
  ```

  `clerk init` does the following for you:

  - Signs you in to Clerk, connects your project to a Clerk application, and pulls your [API keys](/docs/guides/development/clerk-environment-variables) into your `.env.local` file.
  - Installs the `@clerk/nextjs` SDK.
  - Scaffolds Clerk's middleware, `<ClerkProvider>`, and sign-in and sign-up pages.

  > [!TIP]
  > Want to start without signing in? Run `clerk init --keyless` to scaffold your app with temporary development keys, then run `clerk auth login` when you're ready to connect a Clerk account.

  ## Review what `clerk init` set up

  When `clerk init` finishes, your project is ready to authenticate users. Here's what it added:

  - **`.env.local`** — your Clerk [publishable and secret keys](/docs/guides/development/clerk-environment-variables), pulled from the application you connected to.
  - **`proxy.ts`** (or `middleware.ts` on Next.js 15 and earlier) — adds [`clerkMiddleware()`](/docs/reference/nextjs/clerk-middleware) with route protection. By default, every route except `/sign-in` and `/sign-up` requires the user to be signed in. Edit the `isPublicRoute` matcher to change which routes are public.
  - **`app/layout.tsx`** — wraps your app's root layout in [`<ClerkProvider>`](/docs/reference/components/clerk-provider), which makes Clerk's [components](/docs/reference/components/overview) and hooks available throughout your app.
  - **`app/sign-in` and `app/sign-up`** — dedicated pages rendering Clerk's [`<SignIn />`](/docs/reference/components/authentication/sign-in) and [`<SignUp />`](/docs/reference/components/authentication/sign-up) components.
