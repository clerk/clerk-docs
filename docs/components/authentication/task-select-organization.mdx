The `<TaskSelectOrganization />` component renders a UI for resolving the `select-organization` after-auth task. The functionality of the `<TaskSelectOrganization />` component is controlled by the instance settings you specify in the [Clerk Dashboard](https://dashboard.clerk.com), such as [sign-in and sign-up options](/docs/authentication/configuration/sign-up-sign-in-options) and [social connections](/docs/authentication/social-connections/oauth). You can further customize your `<TaskSelectOrganization />` component by passing additional [properties](#properties) at the time of rendering.

> [!IMPORTANT]
> The `<TaskSelectOrganization/>` component cannot render when a user doesn't have current session tasks.

## Properties

All props are optional.

<Properties>
  - `redirectUrlComplete`
  - `string`

  The full URL or path to navigate to after successfully completing all tasks.

  ---

  - `appearance`
  - <code>[Appearance](/docs/customization/overview) | undefined</code>

  Optional object to style your components. Will only affect [Clerk components](/docs/components/overview) and not [Account Portal](/docs/account-portal/overview) pages.
</Properties>

## Usage with frameworks

The following example demonstrates how to host the `<TaskSelectOrganization />` component on a custom page.

<Tabs items={["Next.js", "React"]}>
  <Tab>
    ```tsx {{ filename: 'app/layout.tsx', mark: [7] }}
    import { ClerkProvider } from '@clerk/nextjs'

    export default function RootLayout({ children }: { children: React.ReactNode }) {
      return (
        <html lang="en">
          <body>
            <ClerkProvider taskUrls={{ 'select-organization': '/onboarding/select-organization' }}>
              {children}
            </ClerkProvider>
          </body>
        </html>
      )
    }
    ```

    The `<TaskSelectOrganization />` component must be used in conjunction with the `<SignIn />` component. See the [dedicated guide on how to self-host the `<SignIn />` component](/docs/references/nextjs/custom-sign-in-or-up-page).

    ```tsx {{ filename: 'app/sign-in/[[...sign-in]]/select-organization/page.tsx', mark: [6] }}
    import { TaskSelectOrganization } from '@clerk/nextjs'

    export default function SelectOrganizationPage() {
      return <TaskSelectOrganization redirectUrlComplete="/dashboard" />
    }
    ```
  </Tab>

  <Tab>
    ```tsx {{ filename: 'index.tsx', mark: [17] }}
    import React from 'react'
    import ReactDOM from 'react-dom/client'
    import App from './App.tsx'
    import { ClerkProvider } from '@clerk/clerk-react'

    // Import your Publishable Key
    const PUBLISHABLE_KEY = import.meta.env.VITE_CLERK_PUBLISHABLE_KEY

    if (!PUBLISHABLE_KEY) {
      throw new Error('Add your Clerk Publishable Key to the .env file')
    }

    ReactDOM.createRoot(document.getElementById('root')!).render(
      <React.StrictMode>
        <ClerkProvider
          publishableKey={PUBLISHABLE_KEY}
          taskUrls={{ 'select-organization': '/onboarding/select-organization' }}
        >
          <App />
        </ClerkProvider>
      </React.StrictMode>,
    )
    ```

    ```jsx {{ filename: 'src/onboarding/select-organization.tsx', mark: [3] }}
    import { TaskSelectOrganization } from '@clerk/clerk-react'

    const SelectOrganizationPage = () => <TaskSelectOrganization redirectUrlComplete="/dashboard" />

    export default SelectOrganizationPage
    ```
  </Tab>
</Tabs>

  ## Usage with JavaScript

  The following methods available on an instance of the [`Clerk`](/docs/references/javascript/clerk) class are used to render and control the `<TaskSelectOrganization />` component:

  - [`mountTaskSelectOrganization()`](#mount-task-select-organization)
  - [`unmountTaskSelectOrganization()`](#unmount-task-select-organization)

  The following examples assume that you have followed the [quickstart](/docs/quickstarts/javascript) in order to add Clerk to your JavaScript application.

  ### `mountTaskSelectOrganization()`

  Render the `<TaskSelectOrganization />` component to an HTML `<div>` element.

  ```typescript
  function mountTaskSelectOrganization(
    node: HTMLDivElement,
    props?: TaskSelectOrganizationProps,
  ): void
  ```

  #### `mountTaskSelectOrganization()` params

  <Properties>
    - `node `
    - [`HTMLDivElement`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLDivElement)

    The `<div>` element used to render in the `<TaskSelectOrganization />` component

    ---

    - `props?`
    - [`TaskSelectOrganizationProps`](#properties)

    The properties to pass to the `<TaskSelectOrganization />` component.
  </Properties>

  #### `mountTaskSelectOrganization()` usage

  ```typescript {{ filename: 'main.ts', mark: [26] }}
  import { Clerk } from '@clerk/clerk-js'

  const pubKey = import.meta.env.VITE_CLERK_PUBLISHABLE_KEY

  const clerk = new Clerk(pubKey)
  await clerk.load()

  if (clerk.isSignedIn) {
    // Mount user button component
    document.getElementById('signed-in').innerHTML = `
            <div id="user-button"></div>
          `

    const userbuttonDiv = document.getElementById('user-button')

    clerk.mountUserButton(userbuttonDiv)
  } else if (clerk.session.currentTask) {
    switch (clerk.session.currentTask.key) {
      case 'select-organization': {
        document.getElementById('app').innerHTML = `
                <div id="task-select-organization"></div>
              `

        const taskSelectOrganizationDiv = document.getElementById('task-select-organization')

        clerk.mountTaskSelectOrganization(taskSelectOrganizationDiv)
      }
    }
  }
  ```

  ### `unmountTaskSelectOrganization()`

