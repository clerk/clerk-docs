When building a resource server that needs to accept and verify API keys issued by Clerk, it's crucial to verify these keys on your backend to ensure the request is coming from an authenticated client.

Clerk's Nuxt SDK provides an [`event.context.auth()`](/docs/reference/nuxt/overview#auth-object) function that supports token validation via the `acceptsToken` parameter. This lets you specify which type(s) of token your event handler should accept in your Nuxt application.

By default, `acceptsToken` is set to `session_token`, which means API keys will **not** be accepted unless explicitly configured. You can pass either a **single token type** or an **array of token types** to `acceptsToken`. To learn more about the supported token types, see the [`event.context.auth()` parameters documentation](/docs/reference/nuxt/overview#event-context-auth-options).
