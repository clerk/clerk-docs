## add()

Adds a new domain to the instance. Useful in the case of multi-domain instances, allows adding [satellite domains](/docs/guides/dashboard/dns-domains/satellite-domains) to an instance.

### Parameters

| Parameter | Type                                          |
| --------- | --------------------------------------------- |
| `params`  | [`AddDomainParams`](../add-domain-params.mdx) |

### Returns

`Promise`\<[`Domain`](../domain.mdx)\> — The created [`Domain`](/docs/reference/backend/types/domain) object.

---

## delete()

Deletes a satellite domain for the instance. It is currently not possible to delete the instance's primary domain.

### Parameters

| Parameter           | Type     | Description                               |
| ------------------- | -------- | ----------------------------------------- |
| `satelliteDomainId` | `string` | The ID of the satellite domain to delete. |

### Returns

`Promise`\<[`DeletedObject`](../deleted-object.mdx)\> — The [`DeletedObject`](/docs/reference/backend/types/deleted-object).

## list()

Gets the list of domains for the instance. By default, the list is returned in descending order by creation date (newest first).

### Returns

`Promise`\<[`PaginatedResourceResponse`](../paginated-resource-response.mdx)\<<code>[Domain](../domain.mdx)[]</code>\>\> — A [`PaginatedResourceResponse`](/docs/reference/backend/types/paginated-resource-response) object with a `data` property containing an array of [`Domain`](/docs/reference/backend/types/domain) objects and a `totalCount` property containing the total number of domains for the instance.

---

## update()

Updates a domain for the instance. Both primary and satellite domains can be updated. If you choose to use Clerk via proxy, use this endpoint to specify the `proxy_url`. Whenever you decide you'd rather switch to DNS setup for Clerk, simply set `proxy_url` to `null` for the domain.

When you update a production instance's primary domain name, you have to make sure that you've completed all the necessary setup steps for DNS and emails to work. Expect downtime otherwise. Updating a primary domain's name will also update the instance's home origin, affecting the default application paths.

### Parameters

| Parameter | Type                                                |
| --------- | --------------------------------------------------- |
| `params`  | [`UpdateDomainParams`](../update-domain-params.mdx) |

### Returns

`Promise`\<[`Domain`](../domain.mdx)\> — The updated [`Domain`](/docs/reference/backend/types/domain) object.
