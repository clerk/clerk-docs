Controls whether a satellite app automatically syncs authentication state with the primary domain on first page load. When `true` (default), the satellite app redirects to the primary domain on every first visit to sync state, even if no session cookies exist. When explicitly set to `false`, the satellite app skips the automatic redirect if no session cookies exist, and only triggers the handshake after the user initiates a sign-in or sign-up action. Defaults to `true`. See [satellite domains](/docs/guides/dashboard/dns-domains/satellite-domains) for more details.
