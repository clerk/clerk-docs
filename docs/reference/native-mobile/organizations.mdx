Use the Android organization APIs to create and manage organizations, switch the active organization, and build custom organization flows. Organization methods return `ClerkResult`, so handle both `ClerkResult.Success` and `ClerkResult.Failure`.

```kotlin
import com.clerk.api.network.serialization.ClerkResult
import com.clerk.api.organizations.Organization
import com.clerk.api.organizations.OrganizationDomain
import com.clerk.api.organizations.OrganizationInvitation
import com.clerk.api.user.User

when (val result = Organization.create(name = "Acme", slug = "acme")) {
    is ClerkResult.Success -> {
        val organization = result.value
    }
    is ClerkResult.Failure -> {
        // Show an error state.
    }
}
```

## Active organization

The active organization is stored on `Session.lastActiveOrganizationId` for the current session. Use [`Clerk.organization`](/docs/reference/native-mobile/clerk#active-organization-state) to read the active `Organization` and `Clerk.organizationMembership` to read the signed-in user's active `OrganizationMembership`.

To switch organizations, call `Clerk.auth.setActive(sessionId = sessionId, organizationId = organizationId)`. Pass `organizationId = null`, or omit it, to select the personal account. Selecting a personal account isn't allowed when `Clerk.organizationSelectionIsForced` is `true`.

## Organization

`Organization` represents a Clerk organization and provides helpers for profile, role, membership, invitation, request, and domain management.

### Create or get an organization

```kotlin
when (val result = Organization.create(name = "Acme", slug = "acme")) {
    is ClerkResult.Success -> {
        val organization = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}

when (val result = Organization.get(id = "org_123")) {
    is ClerkResult.Success -> {
        val organization = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}
```

## Memberships

Use membership APIs to list members, add members, update roles, remove members, and check organization permissions.

### List memberships

```kotlin
when (
    val result = organization.getOrganizationMemberships(
        query = "jane",
        role = "org:member",
        limit = 20,
        offset = 0,
    )
) {
    is ClerkResult.Success -> {
        val memberships = result.value.data
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}
```

### Create or remove a membership

```kotlin
when (val result = organization.createMembership(role = "org:member", userId = userId)) {
    is ClerkResult.Success -> {
        val membership = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}

when (val result = organization.removeMember(userId = userId)) {
    is ClerkResult.Success -> {
        val removedMembership = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }

```

### Update or delete a membership

`OrganizationMembership.updateMembership(userId, role)` updates a member's role in the membership's organization. `OrganizationMembership.delete()` deletes the signed-in user's membership in that organization.

```kotlin
when (val result = membership.updateMembership(userId = userId, role = "org:admin")) {
    is ClerkResult.Success -> {
        val updatedMembership = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}

when (val result = membership.delete()) {
    is ClerkResult.Success -> {
        val deletedObject = result.value
    }
    is ClerkResult.Failure -> {
        // Handle the failure.
    }
}
```

### Check membership permissions

`OrganizationMembership` includes `hasPermission(...)` and convenience properties for Clerk system permissions:

- `canManageProfile`
- `canDeleteOrganization`
- `canReadMemberships`
- `canManageMemberships`
- `canReadDomains`
- `canManageDomains`
- `canReadBilling`
- `canManageBilling`
- `canReadApiKeys`
- `canManageApiKeys`

```kotlin
if (membership.canManageMemberships) {
    // Show member management actions.
}

if (membership.hasPermission("org:sys_domains:manage")) {
    // Show domain management actions.
}
```

## API summary

| Resource | APIs |
| - | - |
| `Organization` companion | `create(name, slug?)`, `get(id)` |
| `Organization` profile | `update(name?, slug?)`, `delete()`, `updateLogo(file)`, `deleteLogo()`, `reload()` |
| `Organization` roles | `getRoles(offset, limit)`, `getRolesPaginated(offset, limit)` |
| `Organization` memberships | `getOrganizationMemberships(query?, role?, limit, offset)`, `createMembership(role, userId)`, `removeMember(userId)` |
| `Organization` invitations | `createInvitation(emailAddress, role)`, `getInvitations(limit, offset, status)`, `bulkCreateInvitations(emailAddresses, role)` |
| `Organization` membership requests | `getMembershipRequests(limit, offset, status)` |
| `Organization` domains | `createDomain(name)`, `getDomains(limit, offset, enrollmentMode)`, `getDomain(domainId)`, `deleteDomain(domainId)` |

| `OrganizationMembership` | `hasPermission(...)`, `updateMembership(userId, role)`, `delete()` |
