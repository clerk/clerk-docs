The `<OAuthConsent />` component renders a consent screen for Clerk OAuth provider flows. It loads consent metadata for the signed-in user, displays the OAuth application's identity and requested [scopes](/docs/guides/configure/auth-strategies/oauth/how-clerk-implements-oauth#scopes), and submits the user's allow or deny decision to Clerk.

Use this component when you need to host the OAuth consent page on your own route. For setup instructions and security guidance, see [Customize the OAuth consent page](/docs/guides/configure/auth-strategies/oauth/custom-consent-page).

> [!IMPORTANT]
> The `<OAuthConsent />` component requires a signed-in user. In a normal OAuth authorization flow, Clerk redirects signed-out users to sign-in before sending them to the consent page. If users can visit your custom consent route directly, handle signed-out users the same way you handle any other protected route in your app.

## Properties

<Properties>
  - `appearance?`
  - <code>[Appearance](/docs/guides/customizing-clerk/appearance-prop/overview) | undefined</code>

  An object used to style the component. Do not use `appearance` overrides to hide scopes, redirect warnings, deny actions, or application identity.

  ---

  - `oauthClientId?`
  - `string`

  Overrides the OAuth client ID. Defaults to the `client_id` query parameter from the current URL.

  ---

  - `scope?`
  - `string`

  Overrides the OAuth scope. Defaults to the `scope` query parameter from the current URL.
</Properties>
