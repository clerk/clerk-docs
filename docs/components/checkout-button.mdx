The `<CheckoutButton />` component renders a button that opens the Checkout drawer when clicked. This component allows users to subscribe to plans, either for their personal account or organization. The component must be rendered inside a `<SignedIn />` component to ensure the user is authenticated.

## Properties

<Properties>
  - `planId`
  - `string`

  The ID of the plan to subscribe to.

  ---

  - `planPeriod?`
  - `"month" | "annual"`

  Specify the billing period for the subscription.

  ---

  - `subscriberType?`
  - `"user" | "org"`

  Determines whether the subscription is for the current user or organization. Defaults to `"user"`.

  ---

  - `children?`
  - `React.ReactNode`

  Optional custom button element. If not provided, defaults to a button with "Checkout" text.

  ---

  - `onSubscriptionComplete?`
  - `() => void`

  Optional callback function that is called when a subscription is successfully completed.

  ---

  - `newSubscriptionRedirectUrl?`
  - `string`

  Optional URL to redirect to after a successful subscription.

  ---

  - `checkoutProps?`
  - `object`

  Additional configuration options for the checkout drawer. For example: `<CheckoutButton checkoutProps={{ appearance: { ... } }} />`.
</Properties>

## Usage with frameworks

The following examples demonstrate how to use the `<CheckoutButton />` component in different frameworks.

<Tabs items={["Next.js", "React"]}>
  <Tab>
    ```tsx {{ filename: 'app/pricing/page.tsx' }}
    'use client'

    import { SignedIn } from '@clerk/nextjs'
    import { CheckoutButton } from '@clerk/nextjs/experimental'

    export default function PricingPage() {
      return (
        <SignedIn>
          {/* Basic usage */}
          <CheckoutButton planId="cplan_xxx" planPeriod="month" />

          {/* Custom button with organization subscription */}
          <CheckoutButton
            planId="cplan_xxx"
            planPeriod="annual"
            onSubscriptionComplete={() => {
              console.log('Subscription completed!')
            }}
            newSubscriptionRedirectUrl="/dashboard"
          >
            <button className="custom-button">Subscribe to Pro Plan</button>
          </CheckoutButton>
        </SignedIn>
      )
    }
    ```

## Requirements

The component will throw errors in the following cases:

1. When rendered outside of a `<SignedIn />` component:

```tsx
<>
  // ❌ This will throw an error - A signed in user is required.
  <CheckoutButton planId="cplan_xxx" />
  // ✅ Correct usage
  <SignedIn>
    <CheckoutButton planId="cplan_xxx" />
  </SignedIn>
</>
```

2. When using `subscriberType="org"` without an active organization context:

```tsx
<>
  // ❌ This will throw an error if no organization is active
  <CheckoutButton planId="cplan_xxx" subscriberType="org" />
  // ✅ Correct usage - use `useAuth()` or `await auth()` to grab the `orgId`
  {auth.orgId ? <CheckoutButton planId="cplan_xxx" subscriberType="org" /> : null}
</>
```

## Customization

The component accepts a custom button element as children, allowing you to fully customize its appearance while maintaining the checkout functionality:

```tsx
<CheckoutButton planId="cplan_xxx">
  <button className="custom-button">
    <Icon name="credit-card" />
    Subscribe Now - $9.99/month
  </button>
</CheckoutButton>
```

### Event Handling

The component preserves any click handlers attached to custom button elements while adding the checkout functionality:

```tsx
<CheckoutButton planId="cplan_xxx">
  <button onClick={() => console.log('Starting checkout')} className="custom-button">
    Start Subscription
  </button>
</CheckoutButton>
```
