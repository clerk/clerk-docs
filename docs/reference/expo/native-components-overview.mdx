The `@clerk/expo` SDK provides native UI components that render using SwiftUI on iOS and Jetpack Compose on Android. These components provide a fully native authentication experience while automatically synchronizing with the JavaScript SDK.

## Available components

| Component | Description |
| --- | --- |
| [`<AuthView />`](/docs/reference/expo/auth-view) | Full sign-in and sign-up authentication UI |
| [`<UserButton />`](/docs/reference/expo/user-button-native) | Circular avatar button that opens the user profile |
| [`<UserProfileView />`](/docs/reference/expo/user-profile-view) | Complete profile management interface |

## How it works

The native components handle authentication through the native Clerk SDKs ([clerk-ios](https://github.com/clerk/clerk-ios) and [clerk-android](https://github.com/clerk/clerk-android)). When authentication completes, the session is automatically synchronized to the JavaScript SDK:

1. **Native Authentication** - User authenticates through the native UI
2. **Session Created** - Native SDK creates and manages the session
3. **Session Sync** - `@clerk/expo` syncs the native session to the JS SDK
4. **JS SDK Ready** - All hooks (`useUser`, `useAuth`, `useSession`, etc.) reflect the authenticated state

## Quick example

```tsx {{ filename: 'app/(auth)/sign-in.tsx' }}
import { AuthView } from '@clerk/expo/native'
import { useUser } from '@clerk/expo'
import { useRouter } from 'expo-router'

export default function SignInScreen() {
  const { user } = useUser()
  const router = useRouter()

  // If already signed in, redirect
  if (user) {
    router.replace('/(home)')
    return null
  }

  return (
    <AuthView
      mode="signInOrUp"
      onSuccess={() => router.replace('/(home)')}
    />
  )
}
```

## Requirements

Native components require:

- Expo SDK 53 or later
- Development build (`npx expo run:ios` or `npx expo run:android`)
- The `@clerk/expo` plugin configured in `app.json`:

```json {{ filename: 'app.json' }}
{
  "expo": {
    "plugins": ["@clerk/expo"]
  }
}
```

> [!NOTE]
> Native components are not available in Expo Go. You must use a development build.

## Supported features

The native components support all authentication methods configured in your [Clerk Dashboard](https://dashboard.clerk.com):

- Email/password authentication
- Phone number authentication
- Username authentication
- OAuth providers (Google, Apple, GitHub, Microsoft, etc.)
- Passkeys
- Multi-factor authentication (SMS, TOTP, backup codes)
- Password reset flows
- Enterprise SSO (SAML, OIDC)

## Web compatibility

For Expo Web projects, use the web-specific components from `@clerk/expo/web`:

```tsx {{ filename: 'app/sign-in.web.tsx' }}
import { SignIn } from '@clerk/expo/web'

export default function SignInScreen() {
  return <SignIn />
}
```

See the [component reference](/docs/reference/components/overview) for web component documentation.
