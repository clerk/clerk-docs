Adds a new domain to the instance. Useful in the case of multi-domain instances, allows adding [satellite domains](/docs/guides/dashboard/dns-domains/satellite-domains) to an instance.

Returns the created [`Domain`](/docs/reference/backend/types/domain) object.

```typescript
function add(params: AddDomainParams): Promise<Domain>
```

## `AddDomainParams`


| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="is_satellite"></a> `is_satellite` | `boolean` | Whether the new domain is a satellite domain. Only `true` is accepted at the moment. |
| <a id="name"></a> `name` | `string` | The new domain name. For development instances, can contain the port, e.g. `myhostname:3000`. For production instances, must be a valid FQDN, e.g. `mysite.com`. Cannot contain protocol scheme. |
| <a id="proxy_url"></a> `proxy_url?` | <code>string \| null</code> | The proxy URL for the domain. Applicable only to production instances. |
