title: Just-in-Time (JIT) Provisioning
description: Learn about Clerk's support for Just-in-Time (JIT) Provisioning during SAML SSO.
---

Just-in-Time (JIT) Provisioning is a process by which user accounts are created on-demand the first time an employee authenticates via SAML SSO. It is the simpler of the two provisioning approaches Clerk supports — the alternative is [Directory Sync (SCIM)](/docs/guides/configure/auth-strategies/enterprise-connections/directory-sync), which proactively syncs user lifecycle changes from your IdP independent of sign-in events.

JIT Provisioning requires no additional configuration. Once a SAML enterprise connection is set up, Clerk automatically creates a user account when someone with a matching email domain signs in for the first time. Your IT department won't need to manually create user accounts for each service employees use.

Clerk supports JIT account provisioning for all [supported SAML providers](/docs/guides/configure/auth-strategies/enterprise-connections/overview).

> [!NOTE]
> If you need users to be provisioned and deprovisioned automatically — without waiting for a sign-in event — use [Directory Sync (SCIM)](/docs/guides/configure/auth-strategies/enterprise-connections/directory-sync) instead. This is the recommended approach for organizations that need prompt deprovisioning (e.g., when an employee leaves).

## Sync user attributes during sign in

During SAML SSO and after a user has successfully authenticated, the IdP provides Clerk with the corresponding user data. After each successful sign in, Clerk handles keeping the user data up-to-date based on the response of the SAML provider. This means that if a user's data changes on the IdP side, Clerk will automatically update the user's data in the Clerk database.

To disable this behavior:

1. In the Clerk Dashboard, navigate to the [**SSO connections**](https://dashboard.clerk.com/~/user-authentication/sso-connections) page.
1. Select the SAML connection you want to disable the sync for.
1. Select the **Advanced** tab.
1. Toggle off the **Sync user attributes during Sign in** option.
