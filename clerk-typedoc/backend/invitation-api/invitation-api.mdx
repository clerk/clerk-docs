## createInvitation()

Creates a new invitation for the given email address, and sends the invitation email.

If an email address has already been invited or already exists in your application, trying to create a new invitation will return an error. To bypass this error and create a new invitation anyways, set `ignoreExisting` to `true`.

### Parameters

| Parameter | Type                                   |
| --------- | -------------------------------------- |
| `params`  | [`CreateParams`](../create-params.mdx) |

### Returns

`Promise`\<[`Invitation`](/docs/reference/backend/types/backend-invitation)\> — The newly created [`Invitation`](/docs/reference/backend/types/backend-invitation).

## getInvitationList()

Gets a list of non-revoked invitations for the instance, sorted by descending creation date. By default, the list is returned in descending order by creation date (newest first).

### Parameters

| Parameter         | Type                                                                       | Description                                                                                                                                                                                                                                                                                                                                                                        |
| ----------------- | -------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `params`          | [`GetInvitationListParams`](../get-invitation-list-params.mdx)             | -                                                                                                                                                                                                                                                                                                                                                                                  |
| `params.limit?`   | `number`                                                                   | Maximum number of items returned per request. Must be an integer greater than zero and less than `501`. Can be used for paginating the results together with offset. Defaults to `10`.                                                                                                                                                                                             |
| `params.offset?`  | `number`                                                                   | Skip the first `offset` items when paginating. Needs to be an integer greater or equal to zero. To be used in conjunction with `limit`. Defaults to `0`.                                                                                                                                                                                                                           |
| `params.orderBy?` | `WithSign`\<<code>"created_at" \| "email_address" \| "expires_at"</code>\> | Filters the invitations in a particular order. Prefix a value with `+` to sort in ascending order, or `-` to sort in descending order. Defaults to `-created_at`. Example: `// Newest first await clerkClient.invitations.getInvitationList({ orderBy: '-created_at' }); // Alphabetical by email await clerkClient.invitations.getInvitationList({ orderBy: 'email_address' });`. |
| `params.query?`   | `string`                                                                   | Filters invitations based on `email_address` or `id`. **Example** Get all invitations for a specific email address `import { createClerkClient } from '@clerk/backend'; const clerkClient = createClerkClient(...) await clerkClient.invitations.getInvitationList({ query: 'user@example.com' })`                                                                                 |
| `params.status?`  | <code>"pending" \| "accepted" \| "revoked" \| "expired"</code>             | Filters invitations based on their status. **Example** Get all revoked invitations `import { createClerkClient } from '@clerk/backend'; const clerkClient = createClerkClient(...) await clerkClient.invitations.getInvitationList({ status: 'revoked' })`                                                                                                                         |

### Returns

`Promise`\<[`PaginatedResourceResponse`](../paginated-resource-response.mdx)\<<code>[Invitation](/docs/reference/backend/types/backend-invitation)[]</code>\>\> — A [`PaginatedResourceResponse`](/docs/reference/backend/types/paginated-resource-response) object with a `data` property containing an array of [`Invitation`](/docs/reference/backend/types/backend-invitation) objects and a `totalCount` property containing the total number of invitations.

## revokeInvitation()

Revokes the given invitation.

Revoking an invitation makes the invitation email link unusable. However, it doesn't prevent the user from signing up if they follow the sign up flow.

Only active (i.e., non-revoked) invitations can be revoked.

### Parameters

| Parameter      | Type     | Description                         |
| -------------- | -------- | ----------------------------------- |
| `invitationId` | `string` | The ID of the invitation to revoke. |

### Returns

`Promise`\<[`Invitation`](/docs/reference/backend/types/backend-invitation)\> — The revoked [`Invitation`](/docs/reference/backend/types/backend-invitation).
