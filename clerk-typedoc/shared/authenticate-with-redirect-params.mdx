## Properties

| Property                                                      | Type                                                                                        | Description                                                                                                                                                                                                                                                   |
| ------------------------------------------------------------- | ------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <a id="continuesignin"></a> `continueSignIn?`                 | `boolean`                                                                                   | Whether to continue existing SignIn (if present) or create a new SignIn.                                                                                                                                                                                      |
| <a id="continuesignup"></a> `continueSignUp?`                 | `boolean`                                                                                   | Whether to continue (i.e. PATCH) an existing SignUp (if present) or create a new SignUp.                                                                                                                                                                      |
| <a id="emailaddress"></a> `emailAddress?`                     | `string`                                                                                    | Email address to use for targeting a Enterprise Connection at sign-up                                                                                                                                                                                         |
| <a id="enterpriseconnectionid"></a> `enterpriseConnectionId?` | `string`                                                                                    | -                                                                                                                                                                                                                                                             |
| <a id="identifier"></a> `identifier?`                         | `string`                                                                                    | Identifier to use for targeting a Enterprise Connection at sign-in                                                                                                                                                                                            |
| <a id="legalaccepted"></a> `legalAccepted?`                   | `boolean`                                                                                   | Whether the user has accepted the legal requirements.                                                                                                                                                                                                         |
| <a id="oidcprompt"></a> `oidcPrompt?`                         | `string`                                                                                    | Optional for `oauth_<provider>` or `enterprise_sso` strategies. The value to pass to the [OIDC prompt parameter](https://openid.net/specs/openid-connect-core-1_0.html#:~:text=prompt,reauthentication%20and%20consent.) in the generated OAuth redirect URL. |
| <a id="redirecturl"></a> `redirectUrl`                        | `string`                                                                                    | The full URL or path to the route that will complete the OAuth or SAML flow. Typically, this will be a simple `/sso-callback` route that calls `Clerk.handleRedirectCallback` or mounts the <AuthenticateWithRedirectCallback /> component.                   |
| <a id="redirecturlcomplete"></a> `redirectUrlComplete`        | `string`                                                                                    | The full URL or path to navigate to after the OAuth or SAML flow completes.                                                                                                                                                                                   |
| <a id="strategy"></a> `strategy`                              | <code>[OAuthStrategy](/docs/reference/types/sso#o-auth-strategy) \| "enterprise_sso"</code> | One of the supported OAuth providers you can use to authenticate with, eg 'oauth_google'. Alternatively `enterprise_sso`, to authenticate with Enterprise SSO.                                                                                                |
