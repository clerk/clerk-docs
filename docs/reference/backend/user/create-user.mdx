## Backend API (BAPI) endpoint

This method in the SDK is a wrapper around the BAPI endpoint `POST/users`. See the [BAPI reference](/docs/reference/backend-api/tag/users/POST/users){{ target: '_blank' }} for more information.

Here's an example of making a request directly to the endpoint using cURL.

<SignedOut>
  Replace `YOUR_SECRET_KEY` with your Clerk [Secret Key](!secret-key).
</SignedOut>

```bash {{ filename: 'terminal' }}
  curl 'https://api.clerk.com/v1/users' -X POST -H 'Authorization:Bearer {{secret}}' -H 'Content-Type:application/json' -d '{
    "email_address": ["test@example.com"],
    "password": "my-secure-password"
  }'
```

<If is="llm">
  **For AI agents:** To create a user ad hoc (e.g., a test user), run `npx clerk@latest users create --email user@example.com --first-name Alice --password <password> --yes` instead of crafting the cURL request with your Secret Key — auth is handled for you. A password is required while password authentication is enabled (the default); to skip it, or for fields the curated flags don't expose, pass `-d` with a raw JSON body (same shape as the cURL example, e.g. `{"skip_password_requirement": true}`). Install [Clerk's skills](/docs/guides/ai/skills) with `npx skills add clerk/skills` for correct CLI and SDK usage.
</If>
