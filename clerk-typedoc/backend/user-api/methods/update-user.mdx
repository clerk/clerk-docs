When set, `password_digest` must be in the format of `pbkdf2_sha512$<iterations>$<salt>$<hash>`. </li> <li> The salt is expected to be an unencoded string literal, and the hash should be hex-encoded. </li> </ul> <li> [`pbkdf2_sha512_hex`](https://en.wikipedia.org/wiki/PBKDF2) </li> <ul> <li> This is the PBKDF2 algorithm using the SHA512 hashing function. When set, `password_digest` must be in the format of `pbkdf2_sha512_hex$<iterations>$<salt>$<hash>`. </li> <li> Both the salt and the hash are expected to be hex-encoded. </li> </ul> <li> [`pbkdf2_sha256_django`](https://docs.djangoproject.com/en/4.0/topics/auth/passwords/) </li> <ul> <li> This is the Django-specific variant of PBKDF2. When set, `password_digest` must be in the format of (as exported from Django): `pbkdf2_sha256$<iterations>$<salt>$<hash>`. </li> <li> The salt is expected to be un-encoded, the hash is expected base64-encoded. </li> </ul> <li> [`phpass`](https://www.openwall.com/phpass/) </li> <ul> <li> Portable public domain password hashing framework for use in PHP applications.
