# Appearance prop

The appearance prop can be applied to [React's `<ClerkProvider/>`](/reference-react/general/clerk-provider) to share styles across every component, or individually to any of the Clerk components.

This applies to all of the React-based packages, like [Next.js](/quickstarts/full-stack/nextjs) and [Gatsby](/quickstarts/full-stack/gatsby), as well as [the pure JavaScript ClerkJS package](/reference-js/clerk/clerk#components).

## `@clerk/themes`

We offer a set of themes that can be used with the `appearance` prop. The themes are available as a package called `@clerk/themes`.

#### Usage

<CodeBlockTabs options={['Next.js', 'React', 'Remix']}>
  ```tsx filename="app.tsx" copy
  import { dark } from '@clerk/themes';
  import { ClerkProvider, SignIn } from '@clerk/nextjs';
  import type { AppProps } from "next/app";

  function MyApp({ Component, pageProps }: AppProps) {
    return (
      <ClerkProvider
        appearance={{
          baseTheme: dark
        }}
      >
        <Component {...pageProps}/>
      </ClerkProvider>
    )
  }

  export default MyApp;
  ```

  ```tsx filename="app.tsx" copy
  import React from "react";
  import "./App.css";
  import { dark } from '@clerk/themes';
  import { ClerkProvider } from "@clerk/clerk-react";

  if (!process.env.REACT_APP_CLERK_PUBLISHABLE_KEY) {
    throw new Error("Missing Publishable Key")
  }
  const clerkPubKey = process.env.REACT_APP_CLERK_PUBLISHABLE_KEY;

  function App() {
    return (
      <ClerkProvider appearance={{
        baseTheme: dark
        }} publishableKey={clerkPubKey}>
        <div>Hello from clerk</div>
      </ClerkProvider>
    );
  }

  export default App;
  ```

## Elements

The elements property is used for fine-grained theme overrides, and is useful when for styling specific HTML elements. You can find the element to target by inspecting the HTML and looking for the class name.

An example would be: `cl-formButtonPrimary` which targets the primary buttons in a Clerk component.

### Usage

```tsx filename="app.tsx" copy
import { ClerkProvider } from '@clerk/nextjs';

<ClerkProvider {...pageProps} appearance={{
  elements: {
    formButtonPrimary: {
      backgroundColor: "#2e026d",
    }
  }
}}>
    {/* ... */}
</ClerkProvider>
```

### Element Styling Methods

Currently Clerk components can be used with the following styling methods:

*   CSS Modules
*   Tailwind CSS
*   Inline CSS Objects
*   Global CSS / CSS libraries

#### CSS Modules

Simply create your Module file and add the CSS you want to apply.

```css filename="styles/SignIn.module.css" copy
.primaryColor {
  background-color: bisque;
  color: black;
}
```

Then you can apply this by importing the file and using the classes whenever required:

```tsx filename="app.tsx" copy
import styles from "../styles/SignIn.module.css";
import { ClerkProvider, SignIn } from "@clerk/nextjs";
import type { AppProps } from "next/app";

function MyApp({ pageProps }: AppProps) {
  return (
    <ClerkProvider {...pageProps}>
      <SignIn
        appearance={{
          elements: {
            formButtonPrimary: styles.primaryColor,
          },
        }}
      />
    </ClerkProvider>
  );
}

export default MyApp;
```

#### Tailwind CSS

```tsx filename="app.tsx" copy
import { ClerkProvider, SignIn } from "@clerk/nextjs";
import type { AppProps } from "next/app";

function MyApp({ pageProps }: AppProps) {
  return (
    <ClerkProvider {...pageProps}>
      <SignIn
        appearance={{
          elements: {
            formButtonPrimary:
              "bg-slate-500 hover:bg-slate-400 text-sm normal-case",
          },
        }}
      />
    </ClerkProvider>
  );
}

export default MyApp;
```

#### Inline CSS Objects

```tsx filename="app.tsx" copy
import { ClerkProvider, SignIn } from "@clerk/nextjs";
import type { AppProps } from "next/app";

function MyApp({ pageProps }: AppProps) {
  return (
    <ClerkProvider {...pageProps}>
      <SignIn
        appearance={{
          elements: {
            formButtonPrimary: {
              fontSize: 14,
              textTransform: "none",
              backgroundColor: "#611BBD",
              "&:hover, &:focus, &:active": {
                backgroundColor: "#49247A",
              },
            },
          },
        }}
      />
    </ClerkProvider>
  );
}

export default MyApp;
```

#### Global CSS

If you prefer you can use a global CSS file to style Clerk components. For example we can override the form button primary styles.

```css filename="styles/global.css" copy
.cl-formButtonPrimary {
  font-size: 14px;
  text-transform: none;
  background-color: #611bbd;
}

.cl-formButtonPrimary:hover,
.cl-formButtonPrimary:focus,
.cl-formButtonPrimary:active {
  background-color: #49247a;
}
```

### CSS libraries

Instead of using the Clerk-provided classes (the cl- prefixed ones), you can pass the classnames provided by the library into the elements config of the appearance prop.

For example, you can apply Bootstrap button and text utility classes to the primary form button like so:

```tsx filename="app.tsx" copy
import { ClerkProvider, SignIn } from "@clerk/nextjs";
import type { AppProps } from "next/app";

function MyApp({ pageProps }: AppProps) {
  return (
    <ClerkProvider {...pageProps}>
      <SignIn
        appearance={{
          elements: {
            formButtonPrimary: "btn btn-info text-capitalize",
          },
        }}
      />
    </ClerkProvider>
  );
}

export default MyApp;
```
