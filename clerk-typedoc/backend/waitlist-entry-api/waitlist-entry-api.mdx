## create()

Create a waitlist entry for the given email address. If the email address is already on the waitlist, no new entry will be created and the existing waitlist entry will be returned.

### Parameters

| Parameter | Type                                                               |
| --------- | ------------------------------------------------------------------ |
| `params`  | [`WaitlistEntryCreateParams`](../waitlist-entry-create-params.mdx) |

### Returns

`Promise`\<[`WaitlistEntry`](../waitlist-entry.mdx)\> — The created or existing [`WaitlistEntry`](/docs/reference/backend/types/backend-waitlist-entry) object.

---

## createBulk()

Creates multiple waitlist entries for the given email addresses. If an email address is already on the waitlist, no new entry will be created and the existing waitlist entry will be returned.

### Parameters

### Returns

`Promise`\<<code>[WaitlistEntry](../waitlist-entry.mdx)[]</code>\> — An array of created or existing [`WaitlistEntry`](/docs/reference/backend/types/backend-waitlist-entry) objects.

## invite()

Invites the given waitlist entry.

### Parameters

| Parameter                | Type                                         | Description                                                    |
| ------------------------ | -------------------------------------------- | -------------------------------------------------------------- |
| `id`                     | `string`                                     | The waitlist entry ID.                                         |
| `params`                 | <code>\{ ignoreExisting?: boolean; \}</code> | Optional parameters for inviting the waitlist entry.           |
| `params.ignoreExisting?` | `boolean`                                    | Whether to ignore an existing invitation. Defaults to `false`. |

### Returns

`Promise`\<[`WaitlistEntry`](../waitlist-entry.mdx)\> — The invited [`WaitlistEntry`](/docs/reference/backend/types/backend-waitlist-entry) object.

---

## list()

Gets a list of waitlist entries for the instance. By default, the list is returned in descending order by creation date (newest first).

### Parameters

| Parameter         | Type                                                                       | Description                                                                                                                                                                            |
| ----------------- | -------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `params`          | [`WaitlistEntryListParams`](../waitlist-entry-list-params.mdx)             | -                                                                                                                                                                                      |
| `params.limit?`   | `number`                                                                   | Maximum number of items returned per request. Must be an integer greater than zero and less than `501`. Can be used for paginating the results together with offset. Defaults to `10`. |
| `params.offset?`  | `number`                                                                   | Skip the first `offset` items when paginating. Needs to be an integer greater or equal to zero. To be used in conjunction with `limit`. Defaults to `0`.                               |
| `params.orderBy?` | `WithSign`\<<code>"created_at" \| "invited_at" \| "email_address"</code>\> | Filters waitlist entries in a particular order. Prefix a value with `+` to sort in ascending order, or `-` to sort in descending order. Defaults to `-created_at`.                     |
| `params.query?`   | `string`                                                                   | Filters waitlist entries by `email_address` or `id`.                                                                                                                                   |
| `params.status?`  | <code>"pending" \| "invited" \| "completed" \| "rejected"</code>           | Filters waitlist entries by status.                                                                                                                                                    |

### Returns

`Promise`\<[`PaginatedResourceResponse`](../paginated-resource-response.mdx)\<<code>[WaitlistEntry](../waitlist-entry.mdx)[]</code>\>\> — A [`PaginatedResourceResponse`](/docs/reference/backend/types/paginated-resource-response) object with a `data` property containing an array of [`WaitlistEntry`](/docs/reference/backend/types/backend-waitlist-entry) objects and a `totalCount` property containing the total number of waitlist entries for the instance.
