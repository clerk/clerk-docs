Learn how to integrate Clerk into your Hono backend for secure user authentication and management. This guide focuses on backend implementation and requires a [Clerk frontend SDK](/docs#explore-by-frontend-framework) to function correctly.

<Steps>
  ## Install `@clerk/hono`

  The [Clerk Hono SDK](/docs/reference/hono/overview) provides a range of backend utilities to simplify user authentication and management in your application.

  Run the following command to install the SDK:

  ```npm
  npm install @clerk/hono
  ```

  ## Set your Clerk API keys

  <Include src="_partials/quickstarts/set-clerk-api-keys" />

  ```sh {{ filename: '.env' }}
  CLERK_PUBLISHABLE_KEY={{pub_key}}
  CLERK_SECRET_KEY={{secret}}
  ```

  ## Add `clerkMiddleware()` to your app

  The [`clerkMiddleware()`](/docs/reference/hono/clerk-middleware) function checks the request's cookies and headers for a session JWT and makes the [`Auth`](/docs/reference/backend/types/auth-object){{ target: '_blank' }} object available through [`getAuth()`](/docs/reference/hono/get-auth).

  ```ts {{ filename: 'index.ts', mark: [2, 7] }}
  import { clerkMiddleware } from '@clerk/hono'
  import { Hono } from 'hono'

  const app = new Hono()

  app.use('*', clerkMiddleware())

  app.get('/', (c) => {
    return c.json({ message: 'Hello!' })
  })

  export default app
  ```

  ## Protect your routes using `getAuth()`

  The [`getAuth()`](/docs/reference/hono/get-auth) helper retrieves the current user's authentication state from the Hono context. It returns the [`Auth` object](/docs/reference/backend/types/auth-object){{ target: '_blank' }}.

  <Include src="_partials/hono/get-auth" />
</Steps>
