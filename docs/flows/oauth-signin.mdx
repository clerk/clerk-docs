## Quick install

Create a new React project and set up Clerk to enable OAuth sign-in with your selected provider and configuration.

```bash {{ prettier: false }}
npx create-next-app@latest my-app --typescript --tailwind --eslint
cd my-app
npm install @clerk/nextjs
```

## Project

View the full project code for this example here, using Clerk's React and OAuth components.

```tsx {{ prettier: false }}
import { SignIn } from '@clerk/nextjs'

export default function SignInPage() {
  return (
    <div className="flex justify-center items-center min-h-screen">
      <SignIn 
        appearance={{
          elements: {
            formButtonPrimary: 'bg-blue-600 hover:bg-blue-700 text-sm normal-case'
          }
        }}
        fallbackRedirectUrl="/"
        forceRedirectUrl="/"
      />
    </div>
  )
}
```

## Before you start

Before you install and use OAuth sign-in, make sure you have:

- A Clerk application in the Dashboard
- At least one OAuth provider enabled (e.g., Google, GitHub, Microsoft) - Set up under the Social Connections section
- Frontend API key and Secret Key available (You'll find these in the API Keys section of the Clerk Dashboard)
- Node.js 16+ and your preferred framework configured
- Make sure you've followed your framework's setup guide

## Usage

Insert the sign-in UI where it belongs in your app and add the following options and styling as needed:

```tsx {{ prettier: false }}
import { SignIn } from '@clerk/nextjs'
import { dark } from '@clerk/themes'

export default function Page() {
  return (
    <SignIn
      appearance={{
        baseTheme: dark,
        elements: {
          formButtonPrimary: 'bg-slate-500 hover:bg-slate-600 text-sm',
          card: 'bg-slate-800'
        }
      }}
      fallbackRedirectUrl="/dashboard"
      forceRedirectUrl="/onboarding"
    />
  )
}
```

## Configuration options

### OAuth providers

Configure which OAuth providers are available for sign-in:

- **Google**: Most popular option with seamless user experience
- **GitHub**: Perfect for developer-focused applications  
- **Microsoft**: Ideal for enterprise and business applications
- **Facebook, Twitter, LinkedIn**: Additional social login options
- **Custom providers**: Set up your own OAuth provider

### Sign-in modes

- **Modal**: Opens sign-in in an overlay (recommended for better UX)
- **Redirect**: Navigates to a dedicated sign-in page

### Fallback options

- **Email/password**: Traditional sign-in method as backup
- **Phone/SMS**: Alternative verification method
- **Magic links**: Passwordless sign-in via email

## OAuth flow details

When a user clicks "Sign in with [Provider]":

1. **Redirect to provider**: User is redirected to the OAuth provider's authorization server
2. **User consent**: User grants permission for your app to access their basic profile information  
3. **Authorization code**: Provider redirects back with an authorization code
4. **Token exchange**: Clerk exchanges the code for access tokens securely on the backend
5. **User creation/sign-in**: User account is created or signed in based on the provider data
6. **Session establishment**: Clerk creates a secure session for the authenticated user

## Security considerations

Clerk handles OAuth security best practices automatically:

- **State parameter validation** to prevent CSRF attacks
- **PKCE (Proof Key for Code Exchange)** for enhanced security
- **Secure token storage** and automatic refresh
- **Scope limitation** to request only necessary permissions
- **Provider verification** to ensure authentic responses

## Common use cases

### Single provider sign-in

For apps that only need one OAuth provider:

```tsx
<SignInButton mode="modal" strategy="oauth_google">
  <button>Sign in with Google</button>
</SignInButton>
```

### Multiple provider options

Display multiple OAuth providers with fallback:

```tsx
<SignIn 
  appearance={{
    elements: {
      socialButtonsBlockButton: 'flex items-center justify-center gap-2'
    }
  }}
/>
```

### Custom styling and branding

Match your application's design:

```tsx
<SignIn
  appearance={{
    variables: {
      colorPrimary: '#0F172A',
      colorBackground: '#ffffff',
      borderRadius: '8px'
    },
    elements: {
      card: 'shadow-lg',
      formButtonPrimary: 'bg-blue-600 hover:bg-blue-700'
    }
  }}
/>
```
