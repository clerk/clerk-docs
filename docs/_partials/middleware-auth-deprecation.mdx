> [!IMPORTANT]
> Auth checks inside `clerkMiddleware()` are no longer recommended. **Protect access as close to the resource as possible**, in the page, Route Handler, Server Action, or other server-side code that reads or mutates the data. See [the guide on protecting content from unauthenticated users](/docs/guides/secure/protect-content#migrating-from-middleware-based-auth-checks).
>
> Keep `clerkMiddleware()`: it's required for Clerk configuration and is still the right place for redirects, headers, and other request handling.
